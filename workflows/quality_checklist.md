# 품질 체크리스트 — STM32 고급 실무교육 (8영역)

## 1. 구조 및 형식
- [ ] HTML 형식으로 작성되었는가?
- [ ] `book_style.css`가 링크되어 있는가?
- [ ] 챕터 도입부에 학습 목표 3~5개가 있는가? ("~할 수 있다" 형태)
- [ ] 프로젝트 버전 업그레이드 정보가 명시되어 있는가?
- [ ] Ch04 이상: 아키텍처 위치 섹션(레이어/인터페이스/FSM/시퀀스)이 있는가?
- [ ] 각 절이 도입 → 개념 → 비유 → 예시 → 실습 → 정리 흐름을 따르는가?
- [ ] 챕터 끝에 핵심 정리 + 연습문제가 있는가?
- [ ] 다음 챕터로의 연결 문장이 있는가?

## 2. 콘텐츠 품질
- [ ] 각 절 분량이 2000~4000자인가?
- [ ] 새 개념 도입 시 비유/실생활 예시가 포함되어 있는가?
- [ ] 기술 용어 첫 등장 시 괄호 안 한글 설명이 있는가?
- [ ] 이전 챕터 지식만으로 이해 가능한가?
- [ ] 이전 챕터 코드를 기반으로 기능 추가하는 누적 성장 방식인가?
- [ ] 한 절에 새로운 개념이 3개 이하인가?

## 3. 코드 품질
- [ ] 모든 코드가 완전하고 NUCLEO-F411RE에서 실행 가능한가?
- [ ] STM32 HAL API를 정확히 사용하는가?
- [ ] 모든 코드에 `LOG_D` / `LOG_I` / `LOG_W` / `LOG_E`가 적용되어 있는가?
- [ ] 한국어 주석이 포함되어 있는가?
- [ ] 코딩 스타일이 일관적인가? (4칸 들여쓰기, 스네이크 케이스)
- [ ] 코드 블록이 30줄 이하인가?

## 4. 시각 자료
- [ ] ASCII art가 사용되지 않았는가? (SVG만 허용)
- [ ] 모든 다이어그램이 SVG 형식인가?
- [ ] SVG 파일이 `figures/` 디렉토리에 저장되어 있는가?
- [ ] 네이밍 규칙을 따르는가? (`chNN_secNN_설명.svg`)
- [ ] 색상 팔레트를 준수하는가? (HAL파랑/Driver초록/Service노랑/App빨강)
- [ ] Ch04 이상: 아키텍처 레이어 다이어그램 SVG가 있는가?

## 5. 특수 코너 (5종 aside)
- [ ] 💡 실무 팁 (`class="tip"`)
- [ ] ❓ 수강생 단골 질문 (`class="faq"`)
- [ ] 🎯 면접 포인트 (`class="interview"`)
- [ ] 🔍 스스로 점검 (`class="metacognition"`)
- [ ] 📌 강사 꿀팁 (`class="instructor-tip"`)

## 6. 실습 및 연습문제
- [ ] 챕터당 최소 1개 직접 실행 가능한 실습이 있는가?
- [ ] 실행 방법 및 예상 결과가 제시되어 있는가?
- [ ] 블룸 분류체계 최소 3개 수준을 포함하는가?

## 7. 심리적 품질
- [ ] 첫 실습이 "반드시 성공"하도록 설계되어 있는가?
- [ ] 학습 불안 유발 구간에 안심 장치가 있는가?
- [ ] 감정 곡선: 호기심 → 불안 → 이해 → 성취감 흐름인가?
- [ ] 실패 정상화 문구가 있는가?
- [ ] 메타인지 촉진 장치가 있는가?

## 8. 강의 적합성
- [ ] 설명을 4시간 강의에서 그대로 쓸 수 있는 수준인가?
- [ ] 비유가 STM32 기술 사실과 정확히 매핑되는가?
- [ ] 수강생 막힘 포인트가 사전 해소되어 있는가?
