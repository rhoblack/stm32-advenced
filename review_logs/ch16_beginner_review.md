## 절별 이해도 평가

### 1절: MCU-PC 통신 프로토콜 설계 원칙

**이해도**: ⭐⭐⭐⭐⭐

- "수도 계량기 두 얼굴" 비유가 텍스트 vs 바이너리를 직관적으로 설명함.
- 텍스트 vs 바이너리 비교 테이블이 명확함.
- "프레임 설계 원칙 3가지"가 체계적으로 정리되어 있음.
- CLI와 대시보드가 같은 UART 포트를 쓴다는 것이 FAQ에서 명확히 해소됨.

### 2절: 시리얼 프레임 구조와 CRC-16 구현

**이해도**: ⭐⭐⭐⭐

- 프레임 구조 ASCII 도표(주석 형태)가 이해를 도움.
- "계산기 영수증" CRC 비유가 좋음.
- CRC 코드 자체는 이해하기 어려울 수 있음 — "어떻게 쓰는가"에 집중하라는 안내가 있어서 심리적 부담이 줄음.
- "CRC 계산 범위에서 STX/ETX를 왜 제외하는가" 설명이 명확함.

**아쉬운 점**: CRC 코드의 이중 for 루프를 처음 보는 독자에게는 여전히 무섭게 느껴질 수 있음.
"이 함수는 블랙박스로 써도 됩니다" 메시지를 코드 상단에 한 줄 추가하면 더 좋을 것 같음.

### 3절: protocol_app.c 직렬화 구현

**이해도**: ⭐⭐⭐⭐⭐

- `build_frame()` 6단계 설명이 코드와 1:1 매핑되어 따라가기 쉬움.
- `idx` 변수 역할 설명(배열 인덱스 + 완성 길이 겸용)이 명확함.
- `memcpy`로 float 직렬화하는 이유 설명 — "캐스팅 대신" 명시가 좋음.
- HAL_BUSY 반환 처리 FAQ가 실무적 의문을 해소해줌.
- `HAL_Delay(20)` 사용 이유를 실무 팁에서 설명 — 납득됨.

### 4절: pyserial 수신 스레드 코드 분석

**이해도**: ⭐⭐⭐⭐

- "코드 분석 수준"임을 서두에 명확히 선언 — 불안감 해소.
- "편의점 직원 2명" 스레드 비유가 이해를 도움.
- FrameParser 상태 전이 요약이 FSM 복습처럼 느껴져서 자연스러움 (Ch03 내용 연결).

**아쉬운 점**: QThread를 처음 보는 독자에게는 `run()` 메서드가 "왜 자동 호출되는가"가 궁금할 수 있음.
한 줄 설명을 코드 주석에 추가하면 좋을 것 같음.

### 5절: PyQt5 실시간 시각화 커스터마이징

**이해도**: ⭐⭐⭐⭐

- "환경 문제로 막힐 때 대안 경로(MCU 로그로 먼저 검증)" 안내가 매우 유용함.
- pip 명령어가 코드 블록으로 제공되어 따라하기 쉬움.
- "그래프가 안 움직일 때" FAQ가 실제로 만날 문제를 정확히 다룸.

### 6절: 마일스톤 ② 전체 아키텍처 완성본 리뷰

**이해도**: ⭐⭐⭐⭐⭐

- Ch03 초안 → v1.6 비교 테이블이 성취감을 명확히 시각화함.
- 마일스톤 ② 달성 기능 체크리스트가 "내가 여기까지 왔구나"를 느끼게 함.
- 전체 아키텍처 SVG가 레이어별 색상 구분으로 한눈에 파악 가능.
