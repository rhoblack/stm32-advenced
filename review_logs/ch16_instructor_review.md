### 막힘 1: "MCU와 Python의 엔디언이 맞지 않으면 어떻게 되나요?"

현장에서 자주 나오는 질문입니다. 본문에 "리틀 엔디언" 명시 있음. ✅
그러나 "엔디언이 실제로 틀렸을 때 어떤 증상이 나오는가"를 보여주면 더 좋습니다.

**권장**: aside.tip에 추가: "엔디언이 틀리면 온도가 -3.4e38 등 이상값으로 표시됩니다. 이 경우 Python struct의 `<`를 `>`로 바꿔 보세요."

### 막힘 2: "CRC16이 0이면 전송 안 해도 되는 건가요?"

실제로 CRC가 우연히 0이 될 수 있습니다. 이 경우 프레임은 정상입니다.
0이 "오류 없음"을 의미하는 게 아니라 "계산 결과"임을 강조 필요.

**권장**: 2절 CRC 설명에 한 줄 추가: "CRC 값 0x0000은 오류가 없다는 의미가 아니라 계산 결과값입니다."

### 막힘 3: "빅 엔디언으로 CRC를 보내는 이유가 있나요?"

업계 관례(Modbus RTU, NMEA 등)에서 CRC를 빅 엔디언으로 전송하는 경우가 많습니다.
Python 파서에서 `(frame[-3] << 8) | frame[-2]`가 빅 엔디언 읽기임을 명확히.

### 막힘 4: "VS Code에서 PyQt5 실행이 안 돼요"

Windows에서 pyqtgraph와 PyQt5 버전 충돌이 흔합니다.
권장 버전: PyQt5==5.15.x, pyqtgraph==0.13.x, Python 3.9~3.11.

**권장**: 5절 환경 설정 코드에 버전 고정 설치 명령 추가:
```bash
pip install pyserial==3.5 PyQt5==5.15.9 pyqtgraph==0.13.3
```
