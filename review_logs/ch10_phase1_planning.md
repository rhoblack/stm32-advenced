- **Title**: Ch10. 스텝 모터(ULN2003) 제어

### 1.1 학습 목표 정의 (블룸 분류)

| 번호 | 목표 | 블룸 수준 |
|------|------|---------|

### 1.2 누적 성장 연결점

| 요소 | Ch09에서 | Ch10에서 | 통합 시나리오 |
|------|---------|---------|-------------|
| RTC | 시간 데이터 수집 | 시침 각도 계산 | 매초 RTC 인터럽트 → 각도 변환 → 모터 구동 |

### 2.2 각 섹션의 학습 경로 (감정 곡선)

```
호기심 →  불안  →  이해  → 성취감 → 자부심
─────────────────────────────────
§1: "스텝 모터?"  (생소함)           ↑
§2: 회로 원리      (약간 어려움)      │

§4: 가속/감속      (안정성 필요)      │
```

**심리 설계**:
- **§1 도입**: "기어비 64:1"이라는 수치로 즉시 신뢰도 확보

- **§5 정점**: RTC 시각과 침이 정확히 일치하는 순간 → 마일스톤 달성감

## 6. 아키텍처 위치 섹션 계획 (도입부)

### 6.1 도입부 구조 (§0)

```
①  아키텍처 위치
    - v1.0 전체 레이어 다이어그램 (HAL → Driver → Service → App)
    - Motor Driver 위치: Driver 레이어 (stepper_driver.c)
    - Clock Service 위치: Service 레이어 (clock_service.c) ← **새로운 레이어 활용**

②  인터페이스 설계
    - stepper_driver.h (공개 함수):
      * void stepper_init(void);
      * void stepper_move_steps(int16_t steps, uint16_t speed_rpm);
      * void stepper_home(void);  // 0° 위치로 복귀
    - clock_service.h (공개 함수):
      * void clock_service_init(void);
      * void clock_service_on_rtc_tick(void);  // RTC 콜백에서 호출
      * uint16_t clock_service_get_motor_position(void);

③  FSM 설계
    - Motor FSM: IDLE → MOVING → (HOMING) → IDLE / ERROR
    - 상태 전이 조건: speed_rpm (0 = IDLE), destination step 도달 등

④  시퀀스 다이어그램
    - RTC_Tick (1초) → RTC Callback
    → clock_service_on_rtc_tick()
    → stepper_move_steps(계산된_steps)
    → Motor 회전 (비블로킹)
```

### 6.2 "v1.0 마일스톤" 강조

- **이전**: v0.9까지는 개별 기능 (RTC, Timer, UART 등) 학습
- **Ch10 의미**: **첫 번째 완전한 응용 프로젝트 완성** (시계 동작)
- **다음**: v1.1부터 센서/디스플레이 추가 (지원 기능 확대)
