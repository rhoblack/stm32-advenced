## 감정 곡선 평가

### 호기심 단계 (§8.0): ⭐⭐⭐⭐⭐ 우수
- "스톱워치의 꿈" 제목이 구체적이고 친근함. Ch07 성과를 먼저 확인("v0.7의 tim_driver — 여러분이 직접 만든 첫 타이머 드라이버입니다")하여 **이전 성취를 인정**하는 문장이 적절함.
- 강사 꿀팁의 핸드폰 스톱워치 시연 제안이 매우 효과적 — 학습자가 "이걸 내가 만든다고?" 호기심 유발.
- 메타인지 박스(스스로 점검)로 Ch07/Ch03 선행 지식 자가 점검 제공 — **학습 준비 상태 확인** 기능 우수.

### 불안 단계 완화 장치 (§8.1~8.2): ⭐⭐⭐⭐ 양호
- **Dead-time 불안 완화**: "원리는 알되, 도구를 쓴다" 메시지가 명확. CubeMX에 위임하는 전략으로 DTG 비트맵 복잡성을 우회함. "실무에서도 CubeMX로 설정합니다" 안심 문구 존재.
- **시소 비유**: 직관적이고 한계점까지 명시. Complementary PWM 개념 전달에 효과적.
- **에어록 비유**: Dead-time의 위험성을 극적으로 전달하되, "MOSFET을 태운 경험은 통과의례" 표현으로 실패를 정상화.
- **개선 필요**: §8.1 → §8.2 전환 시 "출력에서 입력으로의 관점 전환"이 학습자에게 혼란을 줄 수 있으나, §8.3 도입부에서 "관점을 180도 전환합니다"라고 명시적으로 전환 신호를 줌 — 양호.

### 이해 단계 진입 (§8.3~8.4): ⭐⭐⭐⭐ 양호
- Input Capture의 "스톱워치 랩 버튼" 비유가 프로젝트(스톱워치)와 직결되어 학습 동기 유지에 효과적.
- OC/One-Pulse는 "개념 수준"으로 축소하여 인지 부하 관리 성공. 강사 꿀팁에서도 "넘어가세요"로 명시.
- Master-Slave "부장-대리" 비유는 직관적이나 한계(단방향) 명시 우수.

### 도전 단계 지지 (§8.6): ⭐⭐⭐⭐⭐ 우수
- "이 챕터의 핵심 성취 구간에 왔습니다" — **명시적 중요성 선언**이 학습자의 집중도를 높임.
- Ch03 자판기 비유 연결("같은 FSM 패턴, 다른 도메인")이 이전 학습 경험과의 가교 역할.
- 상태 전이표 "빈 칸 채우기" 활동이 능동적 학습 설계의 핵심. 강사 꿀팁에서 15~20분 시간 확보 강조.
- **"빈 칸이 곧 버그"** — 설계의 중요성을 한 문장으로 각인.

### 성취 단계 (§8.7): ⭐⭐⭐⭐ 양호
- switch-case 코드가 상태 전이표와 1:1 대응 — "설계가 코드가 된다" 경험 제공.
- 스켈레톤 제공 전략(강사 꿀팁)으로 30분 내 완성 현실성 확보.
- "Button 누르면 시리얼 모니터에 시간이 올라가는" 시각적 성취감 존재.
- **개선 필요**: 성취 확인 문장이 약함. "축하합니다! Service 레이어 첫 구현을 완성했습니다" 같은 명시적 성취 선언 문구 추가 권고.

## 발견한 코드 버그 (심리적 영향 큼)

### [필수 수정] main.c 버튼 이벤트 분기 오류 (§8.7, 원고 라인 1048~1052)

```c
if (duration >= 2000) {
    stopwatch_handle_event(SW_EVT_BTN_LONG);
} else {
    stopwatch_handle_event(SW_EVT_BTN_LONG);  // ← 버그! SW_EVT_BTN_SHORT여야 함
}
```

**심리적 영향**: 학습자가 이 코드를 그대로 따라 치면 **짧게 눌러도 항상 Reset**이 됨. "내가 뭘 잘못한 거지?"라는 자기 의심 유발. 첫 FSM 구현 실습에서 이런 버그는 **자기효능감에 치명적**. 코드를 보고 "아, 오타였구나"라고 스스로 발견하기 어려운 유형의 버그임.

## 최종 권고

### [필수] 1. main.c 버튼 이벤트 분기 수정
- `else` 분기: `SW_EVT_BTN_LONG` → `SW_EVT_BTN_SHORT`로 수정
- **우선순위**: 최상. 이 버그가 남아 있으면 학습자의 FSM 실습이 완전히 실패함.

### [필수] 2. §8.7 끝에 성취 확인 문장 추가 권고
- "축하합니다! 여러분은 방금 Service 레이어의 첫 번째 모듈을 완성했습니다. Ch03에서 이론으로만 보았던 FSM이 실제로 동작하는 코드가 되었습니다."
- **이유**: Ch08은 FSM 첫 실전 구현이라는 교육적 이정표. 성취 선언 없이 §8.8 통합으로 넘어가면 감정 곡선의 최고점이 묻힘.

### [권장] 3. ISR 컨텍스트 FSM 전이 금지 이유 보강
- 현재 §8.7 실무 팁에서 "중첩 인터럽트 문제가 발생할 수 있습니다"로 간략 처리됨.
- 추가 권고: "ISR 안에서 tim_start()를 호출하면, 또 다른 인터럽트(TIM UEV)가 발생할 수 있습니다. ISR 안의 ISR — 이것이 중첩 인터럽트입니다. 디버깅이 매우 어렵습니다."
- **이유**: 학습 불안 포인트 4번(FSM + 타이머 동기화)의 핵심 우려.

### [권장] 4. §8.8에서 "v0.7 기능 보존" 명시적 선언
- "기존 v0.7의 모든 기능(LED PWM, 주기 콜백)은 변경 없이 동작합니다. 새 코드를 추가했을 뿐, 기존 코드를 수정하지 않았습니다."
- **이유**: 누적 성장 방식에서 "추가하면 기존이 깨지지 않을까?"라는 회귀 불안 해소.

### [권장] 5. §8.5(타이머 연계) 메타인지 박스 추가
