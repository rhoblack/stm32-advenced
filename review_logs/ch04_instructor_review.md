### 시간 위험 분석

**§4.4 (60분)**: 원고 내용 자체는 충분하지만, 학생이 "재등록 패턴"을 처음 접하면 개념 이해에 10~15분 추가 소요 예상. 강사 꿀팁의 "일부러 재등록 빼고 시연" 전략이 매우 효과적이나, 이 시연 + 설명에 5분 추가 필요.

## 3. 라이브 데모 현실성 평가

### Demo 1: SWO vs UART 비교 (도입, 5분)
- **현실성**: 높음
- ST-Link VCP는 Windows에서 자동 인식 (STM32 드라이버 사전 설치 전제)
- **강사 꿀팁** "USB 뽑아보기" 시연이 효과적이나, 별도 전원 공급 준비 필요
- **제안**: 사전 준비된 프로젝트로 시연. USB 뽑기 시연은 선택 사항으로.

### Demo 2: Baud Rate 불일치 (3분)
- **현실성**: 매우 높음
- Tera Term에서 Baud Rate 변경은 즉시 반영 (케이블 재연결 불필요)
- "ÿÿÿ" 깨진 문자 → 115200으로 맞추면 정상 출력: 시각적 임팩트 강함
- **강의에서 가장 기억에 남을 순간** 중 하나

### Demo 3: "Hello UART!" 첫 출력 (15분)
- **현실성**: 높음
- CubeMX 설정은 **강사가 화면 공유하며 같이 진행** 형식 권고
- 최소 1명 완료까지 약 10분 예상 (CubeMX 경험 수준에 따라 편차)
- **필수 준비**: 느린 학생을 위한 사전 완성 CubeMX 프로젝트(.ioc) 제공

### Demo 4: LED 블로킹 체험 (10분)
- **현실성**: 높음
- 폴링 Receive 1초 타임아웃 → LED 끊김 체감: 단순하지만 강력한 시연
- 학생이 직접 체험해야 효과적

### Demo 5: 인터럽트 수신 + LED 제어 (25분)
- **현실성**: 중간~높음
- 단계별 가이드 필요: (1) 콜백 작성 → (2) 초기 등록 → (3) main loop 처리
- **가장 복잡한 실습** — 학생 진도 편차 클 수 있음
- **필수 준비**: "붙여넣기용 코드" 또는 빈칸 채우기 템플릿

### Demo 6: 재등록 빠뜨리기 시연 (5분)
- **현실성**: 매우 높음
- 강사가 일부러 재등록 주석 처리 → 첫 글자만 에코 → "왜?"
- **10초의 실패 시연이 10분의 설명보다 효과적** — 원고의 강사 꿀팁 그대로 활용

## 5. 감정 리듬과 에너지 관리

### 강의 에너지 곡선

```
도입(높음) → §4.1 이론(중간) → §4.2 CubeMX(낮음) → §4.3 첫 출력(높음!)
→ 휴식 → §4.4 인터럽트(높음→난관→성취) → 휴식 → §4.5 리팩토링(중간) → 마무리
```

- **도입**: SWO vs UART 비교가 호기심을 유발. "이미 했는데 왜 또?" 제목이 좋음.
- **§4.1~4.2**: 이론 구간이 연속 50분. **에너지 저점 위험**.
  - **대응 제안**: §4.1 중간에 Demo 2(Baud Rate 불일치)를 삽입하여 이론 구간을 분할.
- **§4.3**: "Hello UART!" 첫 출력 성공이 **가장 큰 성취감 순간**. 이 순간을 충분히 축하.
- **§4.4**: 재등록 패턴이 난관이나, 실패 시연→해결의 드라마틱한 흐름이 좋음.
- **§4.5**: 리팩토링은 에너지가 낮을 수 있음. 완성 코드 제공으로 부담 완화.

## 7. 순조로운 부분 (그대로 강의에 활용 가능)

1. **호텔 모닝콜 비유** (§4.4): 1바이트 재등록 패턴을 한 번에 이해시킬 수 있는 탁월한 비유.
2. **편지 봉투 비유** (§4.1): UART 프레임을 비전공자도 이해할 수 있는 수준으로 풀어냄.
3. **강사 꿀팁 3개**: 모두 실제 강의에서 바로 활용 가능. 특히 "재등록 빼고 시연"은 반드시 실행.
4. **UART 프레임 SVG**: 0x41('A')의 실제 비트 파형을 보여주는 다이어그램이 교과서 수준.
5. **인터럽트 시퀀스 SVG**: 5개 참여자(PC, HW, NVIC, Callback, main loop)의 상호작용이 명확.
6. **면접 포인트 3개**: UART/USART 차이, 폴링 단점, 재등록 패턴 — 실제 면접에서 자주 나오는 질문.

## 8. 조정 필요 사항

- 없음. 원고의 기술적 정확성과 강의 흐름 모두 양호.

### 권장 수정 (3개)

#### R1. §4.5 Driver 분리 — 실습 방식 명시
- **현재**: 학생이 직접 리팩토링하는 것인지, 완성 코드를 보는 것인지 불명확
- **제안**: "강사가 화면 공유하며 리팩토링 과정을 보여주고, 학생은 완성 코드를 받아 빌드·실행만 확인" 형식 명시
- **이유**: 40분 내 학생이 직접 리팩토링하기는 현실적으로 어려움. 코드를 주고 "왜 이렇게 분리했는지"를 토론하는 것이 더 효과적.

#### R2. §4.2 CubeMX 설정 — 시각 가이드 보강
- **현재**: 텍스트 단계별 설명만 있음
- **제안**: "Connectivity → USART2 → Asynchronous" 경로에 대한 **instructor-tip으로 "CubeMX 화면을 프로젝터에 띄우고 학생이 따라하게 하세요. 30초 이상 기다려주세요."** 추가
- **이유**: CubeMX 설정은 학생 진도 편차가 가장 큰 구간. 스크린샷이 이상적이나, instructor-tip으로도 대응 가능.

#### R3. 시간 부족 시 축소 안 (Fallback Plan) 추가
- **현재**: 시간 부족 시 대응 안 없음
- **제안**: 마무리 섹션 또는 강사 가이드에 다음 축소 안 추가:
  - "시간 부족 시: §4.5는 완성 코드만 빌드·실행하고, Driver 분리의 이유만 3분 설명"
  - "§4.4가 길어지면: 재등록 패턴만 확인하고, 링 버퍼 상세는 코드 읽기 과제로"
- **이유**: 4시간 강의에서 10~15분 지연은 흔함. Fallback이 없으면 마무리가 급해짐.
