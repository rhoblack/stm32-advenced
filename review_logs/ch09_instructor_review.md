## 1. 비유 검증 (5개)

### 비유 1: 벽걸이 시계 = 백업 도메인 (§9.1)
- **기술적 정확성**: O — "정전이 되어도 건전지로 계속 돌아간다" = VBAT 기반 독립 동작
- **한계 명시**: O — "NUCLEO에서는 VBAT=VDD라 전원 끄면 소실" 명확히 기술
- **오해 가능성**: 낮음 — VBAT 연결 필요성까지 언급
- **강의 활용**: O — 15분 내 설명 가능, 코인셀 실물 보여주면 효과적
- **판정**: ✅ PASS

### 비유 2: 전광판 숫자 = BCD 레지스터 (§9.1)
- **기술적 정확성**: O — "각 자릿수가 독립적으로 4비트 니블" 정확
- **한계 명시**: O — "0xA~0xF가 나오면 잘못된 값" 설명 있음
- **오해 가능성**: 낮음 — 0x23 ≠ 십진수 35 구분 명시
- **강의 활용**: O — "0x23이 왜 35가 아니라 23?" 한 줄로 핵심 전달
- **판정**: ✅ PASS

### 비유 3: 석영 시계 vs 기계식 시계 = LSE vs LSI (§9.2)
- **기술적 정확성**: O — "정확도 차이"가 핵심 포인트
- **한계 명시**: O — "LSI는 RC 오실레이터이므로 기계식보다 저가형 전자시계에 가깝다" 기술
- **오해 가능성**: 약간 — "기계식"이라 하면 스프링/기어를 연상할 수 있으나, 한계점에서 보정됨
- **강의 활용**: O — NUCLEO는 LSI 기본이라는 점 명확, "프로토타입=LSI, 양산=LSE" 판단 기준 제시
- **판정**: ✅ PASS

### 비유 4: 알람 시계 vs 주방 타이머 = Alarm vs Wake-up (§9.5)
- **기술적 정확성**: O — "특정 시각" vs "일정 주기" 구분 정확
- **한계 명시**: O — "Wake-up은 자동 반복, 주방 타이머와 달리 계속 울린다" 설명
- **오해 가능성**: 낮음 — 표(TIM vs Wake-up)와 FAQ로 이중 보강
- **강의 활용**: O — 두 코드를 나란히 비교하는 구조 제공됨 (§9.4 Alarm + §9.5 Wake-up)
- **판정**: ✅ PASS

### 비유 5: 스마트폰 반복 설정 = 알람 마스크 (§9.4)
- **기술적 정확성**: O — "마스크 ON = 이 필드 무시 = 항상 매칭"
- **한계 명시**: O — "RTC 마스크는 시/분/초/요일 각각 독립 마스킹, 스마트폰보다 유연하지만 복잡"
- **오해 가능성**: 낮음 — 마스크 시나리오 표(5가지)로 구체적 예시 제공
- **강의 활용**: O — 마스크 조합 → 트리거 빈도 표가 칠판 설명에 최적
- **판정**: ✅ PASS

**비유 종합**: 5/5 PASS — 모든 비유에 한계점이 명시되어 있어 오해 위험이 낮음
