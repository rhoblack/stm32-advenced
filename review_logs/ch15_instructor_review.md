## 추가 강사 제안

### 제안 #1: 2절에 "strtok 함정" 라이브 데모 추가 권장

교재에서 strtok의 위험성을 설명하지만, 실제로 strtok을 ISR에서 사용하면
어떤 일이 일어나는지 **라이브 시연**하면 각인 효과가 큽니다.

강의 흐름:
1. 먼저 strtok으로 동작하는 것을 보여줌
2. ISR 시뮬레이션으로 전역 포인터 덮어쓰기 → 오동작 시연
3. strtok_r로 교체 → 정상 동작
4. "이제 여러분은 이 버그를 알게 됐습니다. 인터뷰에서도 나옵니다."

### 제안 #2: 연습문제 3번 (log level 명령어)이 우수한 자기주도 과제

연습문제 3번 "log level N 명령어 추가"는 학습한 커맨드 테이블 패턴을 바로 적용하는 과제입니다.
이 과제가 성공하면 "커맨드 테이블의 확장성을 직접 체험"하는 강력한 학습이 됩니다.
**숙제로 내기에 최적입니다** — 분량도 적당하고 답이 명확합니다.

### 제안 #3: 4절 핸들러 코드 구조가 `switch-case`와 비교되면 더 효과적

현재 4절은 핸들러 코드만 보여줍니다.
`switch-case` 방식으로 먼저 구현 → 문제점(명령어 추가 시 케이스 추가 필요) → 커맨드 테이블로 리팩토링
순서로 진행하면 커맨드 테이블의 장점이 더욱 명확해집니다.
단, 4시간 내에 소화하려면 코드 비교는 슬라이드 1장으로 간략하게 처리하는 것이 좋습니다.
