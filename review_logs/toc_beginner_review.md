## 검토 결과

---

### 🔴 Critical 이슈 (이해가 전혀 안 되는 구간)

**[Ch06] Advanced TIM 심화 — "Basic TIM"이 어디에도 없다**

목차 전체를 훑어봤을 때 타이머(TIM)는 Ch06에서 "Advanced TIM 심화"라는 제목으로 처음 등장한다.
중급 독자라도 DMA처럼 STM32의 TIM 구조(Basic / General / Advanced 계층)는 처음 접하는 상황인데,
사전에 "기본 TIM 개요"나 "Basic/General TIM → Advanced TIM"으로 이어지는 브리지 챕터 또는
도입부 없이 곧바로 Complementary 출력, Dead-time, 타이머 연계(Master-Slave)가 나온다.
제목의 "심화(Advanced)"라는 단어만 봐도 "그럼 기초는 어디서 배웠지?"라는 의문이 즉각 발생한다.
목차만으로는 이 챕터를 수강할 수 있는 선수 지식이 무엇인지 전혀 파악되지 않는다.

**[Ch14] 소프트웨어 아키텍처 설계론 — FSM이 갑작스럽게 "설계 패턴"으로 격상**

FSM(유한상태기계)은 Ch01에서 "예고" 수준으로 언급되고, Ch08에서 모터 상태관리에 활용되다가,
Ch14에서 갑자기 "FSM 설계 패턴 + 상태 전이 다이어그램(SVG)"이라는 본격적인 아키텍처 주제로 등장한다.
목차 상에서 Ch01 → Ch08 → Ch14로 FSM이 어떻게 발전하는지의 흐름이 명시되지 않아서,
Ch14만 놓고 보면 "지금 처음 FSM을 배우는 건지, 이미 아는 걸 정리하는 건지" 판단이 불가능하다.
선수 챕터 참조(예: "Ch01·Ch08 FSM 패턴을 전체 시스템으로 확장") 표기가 필요하다.

### 🟡 Major 이슈 (어렵거나 설명 보강 필요)

**[Ch02] 로그 시스템 — SWO/ITM, SEGGER RTT가 설명 없이 등장**

Ch02는 Ch01 바로 다음에 위치하며 프로젝트의 "표준 인프라"를 구축하는 챕터다.
그런데 SWO/ITM, SEGGER RTT는 STM32에 처음 입문하는 개발자에게 생소한 디버그 인터페이스다.
목차 항목만 봐서는 이게 하드웨어 인터페이스인지, 소프트웨어 라이브러리인지, 별도 장비가 필요한지
전혀 알 수 없다. 짧은 부제(예: "SWO/ITM — 디버거 핀을 통한 printf 출력")만 추가해도 이해도가 크게 오른다.
또한 "링 버퍼 기반 비동기 로그 출력"은 링 버퍼를 아직 배우지 않은 시점에 나오는 개념이다.
링 버퍼의 본격 구현은 Ch05에서 다루므로, Ch02에서의 링 버퍼 언급은 선수 개념 미비로 느껴진다.

**[Ch04] DMA 아키텍처 — "스트림, 채널" 용어 예고 없이 사용**

Ch04가 DMA의 첫 등장 챕터인데도, 목차 핵심 주제 첫 줄에 "스트림, 채널, 우선순위"가 곧바로 나온다.
STM32 DMA의 스트림/채널 개념은 다른 MCU(AVR, 일반 ARM)와 다른 STM32 고유 구조라서
용어 자체에 대한 사전 안내가 목차 수준에서라도 있어야 한다.
예: "DMA 컨트롤러 구조 — STM32 고유의 스트림·채널 개념 이해" 정도의 부연이면 충분하다.

**[Ch07] 내부 RTC — "BCD 레지스터, 백업 도메인, LSE/LSI"가 설명 없이 나열**

RTC를 처음 접하는 독자에게 BCD(Binary-Coded Decimal), 백업 도메인(Backup Domain),
LSE(외부 저속 클럭)/LSI(내부 저속 클럭)는 모두 낯선 용어다.
목차 핵심 주제에 이 셋이 설명 없이 동시에 나열되면, 챕터 수강 전에 무엇을 예습해야 하는지
또는 챕터 안에서 설명해 주는지 알 수 없어 학습 불안감을 유발한다.
적어도 "(백업 도메인: 전원 차단 시에도 유지되는 레지스터 영역)" 같은 한 줄 괄호 설명이 필요하다.

**[Ch10] SPI DMA 통신 — ILI9341을 "처음"으로 SPI에서 배우는 구조**

SPI 통신 자체를 독립 챕터 없이 곧바로 "SPI DMA + ILI9341 TFT LCD"로 진입한다.
UART는 Ch03(기초) → Ch05(DMA 심화)의 2단계 구조인데, SPI는 기초 챕터 없이 DMA 심화가 첫 등장이다.
목차만 봐서는 "SPI 기초는 이미 알고 있어야 하는 건가?"라는 의문이 생긴다.
SPI 기초(HAL_SPI_Transmit, CS핀 수동 제어 등)가 Ch10 안에서 다뤄진다면 목차에 한 줄이라도 명시해야 한다.

**[Ch13] PC 대시보드 — Python/PyQt5가 갑자기 등장, 임베디드 교육 범위 혼선**

STM32 임베디드 교육 과정인데 Ch13에서 Python + pyserial + PyQt5/tkinter가 등장한다.
이 챕터가 "STM32 펌웨어 개발"의 연장선인지, "PC 측 소프트웨어 개발"인지를
목차 수준에서 명확히 구분해 주어야 한다.
대상 독자가 Python 경험이 있다고 가정하는지 여부도 불명확하다.
예: "Python 기초 지식 권장 (리스트, 클래스, 스레드)" 같은 선수 지식 명시가 필요하다.

### 🟢 Minor 이슈 (보완하면 좋은 것)

**[전반] 각 챕터에 "선수 챕터" 또는 "이전 버전 대비 변경 사항" 명시 부재**

누적 성장형(v0.1~v2.0) 구조이므로, 각 챕터 항목에 "(선수: Ch03, Ch04 필수)" 같은 의존 관계를
한 줄이라도 적으면 학습자가 중간 챕터부터 진입할 때 혼선이 줄어든다.

**[Ch08] 스텝 모터 — 하드웨어 회로 지식 선수 요건 불명확**

"ULN2003 드라이버 회로"는 전자회로 지식이 필요한 주제다.
목차 대상 독자 정의(C언어 숙련, HAL 기본 경험)에 회로 지식이 포함되는지 불명확하므로,
"기본 전자회로 독해 가능 수준 권장" 같은 한 줄 안내가 있으면 좋다.

**[Ch11] I2C DMA 심화 — Ch09(I2C 기초)와의 연결 명시 필요**

Ch11이 Ch09의 "DMA 심화편"이라는 관계가 제목에서 보이지 않는다.
"Ch09 SHT31 → DMA 비동기 고속화" 정도의 한 줄 연결 설명이 목차에 있으면 흐름이 명확해진다.
현재는 누적 기능 항목에 "SHT31 → DMA 비동기 고속화"라고 나와 있지만, 핵심 주제 연결이 약하다.

**[Ch15, Ch16] 프로젝트 버전 표기 없음**

Ch01~Ch14는 프로젝트 버전(v0.1~v2.0)이 표기되어 누적 성장 구조가 시각적으로 잘 보이는 반면,
Ch15와 Ch16에는 버전 표기가 없다. "v2.0 완성 이후 품질 향상 단계" 같은 위상 표시가 없으면
이 두 챕터가 선택 과목인지 필수 과목인지 불분명하게 느껴진다.

**[Ch12] CLI — "커맨드 테이블 패턴 (함수 포인터 배열)"의 선수 지식 확인 필요**

함수 포인터는 C 중급 이상의 개념이다. 대상 독자가 "C언어 숙련"이므로 무리한 요구는 아니지만,
함수 포인터를 처음 본다면 이 챕터에서 걸릴 수 있다는 점을 목차에 짧게 안내하면 좋다.

### ✅ 잘 된 점

**누적 성장형(v0.1 → v2.0) 버전 표기 전략이 학습 동기를 명확하게 자극한다**

매 챕터마다 프로젝트 버전 번호와 "누적 기능" 항목이 함께 제시되어,
학습자가 "이 챕터를 마치면 내 프로젝트가 어떻게 달라지는지"를 목차 단계에서 미리 예상할 수 있다.
특히 v1.0(디지털 시계 첫 완성)이라는 중간 마일스톤 설정은 학습 의욕을 유지하는 데 효과적이다.

**UART의 2단계 구조(Ch03 기초 → Ch05 DMA 심화)가 학습 난이도 분산에 좋은 선례를 만든다**

UART를 기초(폴링/인터럽트)와 DMA 심화로 나눠서 챕터를 배치한 구조는
"개념 이해 → 성능 최적화"의 자연스러운 학습 경로를 제공한다.
이 패턴이 I2C(Ch09 → Ch11)에도 일관되게 적용되어 있어서 구조적 일관성이 느껴진다.

**Part 0(기반 구축) → Part 1(통신+DMA) → Part 2(타이머+RTC) → Part 3(센서+통신심화) 순서가 의존성 측면에서 대체로 논리적이다**

하위 레이어(GPIO, 로그)를 먼저 구축하고 통신, 타이머, 센서 순으로 올라가는 구성은
임베디드 시스템 개발의 실제 진행 순서와 유사해서 친숙하게 느껴진다.

**Ch02에서 로그 시스템을 초반에 배치한 것은 실무적으로 매우 탁월한 선택이다**

실제 현업에서도 디버그 출력 인프라를 먼저 구축하고 나머지 기능을 개발하는 방식이 표준인데,
이를 Ch02에 배치해서 "이후 모든 챕터 표준"으로 선언한 것은 실무 감각을 잘 반영한 설계다.

**하드웨어 주변장치(스텝 모터 Ch08, TFT LCD Ch10)를 단순 예제가 아닌 프로젝트 핵심 기능으로 통합한 구성이 좋다**

"스텝 모터 = 아날로그 시침", "TFT LCD = 디지털 시각 + 온습도 표시"처럼
하드웨어 실습이 프로젝트 맥락 안에 자연스럽게 녹아 있어서 학습의 목적성이 명확하다.

**Part 6(디버깅 + 리팩토링)을 마지막 파트로 배치한 구성이 "완성 후 품질 향상"이라는 실무 사이클과 일치한다**

이 배치는 "기능 구현 → 최적화 → 품질 향상"이라는 임베디드 개발 현장의 자연스러운 흐름을
교육과정 구조로 그대로 옮긴 것으로, 학습자에게 실무 감각을 체득하게 하는 데 기여한다.

## 🎓 이해도 점수: 6.5 / 10

**한 줄 총평:**
누적 성장형 구조와 프로젝트 중심 설계는 탁월하나, SPI/TIM/RTC 등 핵심 주제의 선수 개념 안내 부재와 용어 무방비 노출이 중급 독자에게 "이 챕터에서 내가 뭘 모르는지 모르는" 상태를 만들어 학습 진입 장벽을 높이고 있다.
