## 1. 도입부 명확성 — ⭐⭐⭐⭐⭐

**"SWO vs UART" 혼동 해소**: 매우 명확합니다. "청진기 vs 목소리" 비유가 두 기술의 본질적 차이를 한 번에 전달합니다. §4.0 제목부터 "이미 했는데 왜 또?"라고 독자의 의문을 선제적으로 다루는 점이 인상적입니다.

**동기 부여**: "디버거 없이도 동작", "PC에서 MCU로 명령을 보낼 수 있다"는 두 가지 핵심 가치가 명확합니다. Ch01 버튼에 이어 "두 번째 입력 채널"이라는 표현이 누적 성장을 자연스럽게 연결합니다.

**Ch02와의 연결**: SWO → UART 전환이 매우 자연스럽습니다. "ST-Link USB를 뽑아보세요" 강사 꿀팁도 체감 포인트로 적절합니다.

**SWO vs UART SVG** (`ch04_sec01_swo_vs_uart.svg`): 두 경로를 색상으로 명확히 구분(보라=SWO, 초록=UART)하고, 하단 요약 박스가 핵심 차이를 한눈에 보여줍니다. 매우 도움이 됩니다.

## 2. 개념 이해도 — ⭐⭐⭐⭐⭐

### UART 프레임 구조
"편지 봉투" 비유가 Start/Data/Stop과 1:1 대응됩니다. "봉투를 연다 → 편지 읽는다 → 닫는다" 흐름이 직관적입니다. 비유의 한계(편지는 단방향, UART는 Full Duplex)를 명시한 점도 좋습니다.

**UART 프레임 SVG** (`ch04_sec01_uart_frame.svg`): 0x41('A')의 실제 비트 파형을 보여주는 다이어그램이 매우 효과적입니다. LSB First 순서(D0=1부터)가 시각적으로 확인 가능하고, 하단의 "10 bits = 86.8us" 계산이 이론과 그림을 연결합니다.

### Baud Rate
"양쪽이 같은 속도로 약속"이 명확합니다. 불일치 시 "12배 느리게 샘플링 → 깨진 문자" 설명이 구체적이어서 왜 깨지는지 이해할 수 있습니다. "ÿÿÿ" 증상 → 99% Baud Rate 불일치라는 실무 팁이 실용적입니다.

### 115200-8-N-1 해석
"115200 / 10 = 11,520 바이트/초" 계산 과정이 명쾌합니다. 처음 보는 사람도 따라갈 수 있는 수준입니다.

## 3. 폴링 vs 인터럽트 이해 — ⭐⭐⭐⭐

### 폴링 블로킹 체험
"택배 기사 기다리기" 비유가 적절합니다. LED가 500ms가 아닌 ~1.5초로 깜빡이는 구체적 증상이 블로킹의 결과를 체감하게 합니다.

**CPU 타임라인 SVG** (`ch04_sec03_polling_timeline.svg`, `ch04_sec04_polling_vs_interrupt.svg`): 빨간색(블로킹) vs 초록색(여유) 대비가 시각적으로 명확합니다. 특히 `ch04_sec04_polling_vs_interrupt.svg`의 상하 비교가 "인터럽트 = CPU 99% 해방" 메시지를 강력히 전달합니다.

### 약간의 아쉬움
- §4.3의 블로킹 체험 코드에서 `last_led` 변수의 초기화가 코드에 없습니다. `uint32_t last_led = 0;` 선언이 보이지 않아 코드를 그대로 따라 치면 컴파일 에러가 발생합니다. (사소하지만 초급자에게는 혼란 요소)

## 4. 핵심 난관 (§4.4 인터럽트 재등록) — ⭐⭐⭐⭐⭐

### "1바이트 재등록 패턴"의 필요성
**매우 명확합니다.** "왜 한 번에 여러 바이트가 아닌 1바이트인가?"에 대한 FAQ가 핵심을 정확히 짚습니다: "10바이트를 지정하면 정확히 10바이트가 도착해야만 콜백이 호출" — 이 한 문장이 패턴의 존재 이유를 완벽히 설명합니다.

### 호텔 모닝콜 비유
**패턴의 본질을 매우 잘 전달합니다.** "모닝콜은 1회성 → 다음 날도 원하면 다시 예약" = "재등록 패턴"의 핵심을 정확히 비유합니다. 비유의 한계(인터럽트는 "중단", 모닝콜은 "깨어남")도 명시되어 있습니다.

### 시퀀스 다이어그램 → 코드 흐름
**자연스럽습니다.** `ch04_sec04_interrupt_sequence.svg`에서:
1. 초기 등록 → 바이트 도착 → RXNE → ISR → 콜백 → 재등록 순서가 단계별로 표현
2. "Receive_IT 재등록!" 박스가 주황색 강조로 눈에 띔
3. main loop가 "계속 동작" 상태로 표시되어 비블로킹 특성이 시각적으로 확인됨

### 콜백 안의 재등록
코드를 본 후 **"아, 당연하구나"**라는 느낌이 듭니다. 시퀀스 다이어그램 → 단계별 설명(6단계) → 코드 순서가 이해도를 점진적으로 쌓아올립니다. "이 줄이 없으면 첫 1바이트만 수신" + "디버깅에 30분 이상" 경고가 실수를 미연에 방지합니다.

**강사 꿀팁("일부러 재등록 빼고 시연")**이 매우 효과적인 교수법입니다.

## 5. 누적 성장 체감 — ⭐⭐⭐⭐⭐

### Ch01 → Ch04 패턴 인식
- Ch01 EXTI 인터럽트 → Ch04 UART 인터럽트: "Ch01에서 버튼 EXTI 인터럽트를 사용했던 것을 기억하세요"라는 명시적 연결이 있습니다. 동일한 인터럽트 원리를 다른 주변장치에 적용하는 패턴을 인식할 수 있었습니다.

### Ch02 → Ch04 전환

### Ch03 → Ch04 아키텍처 실현
- "회색 블록이 초록색으로 칠해집니다"라는 표현이 Ch03의 설계가 Ch04에서 구현으로 전환되는 느낌을 강하게 줍니다. Driver 분리 체크리스트가 Ch03 이론의 실제 적용을 확인하게 해줍니다.

## 6. 코드 따라하기 — ⭐⭐⭐⭐

### 단계적 진행
폴링 송신(`uart_send_string`) → printf 리다이렉트(`_write`) → 인터럽트 수신(콜백+재등록) → Driver 분리(`uart_driver.c/h`) 순서가 복잡도 증가 형태로 자연스럽습니다.

### CubeMX 설정
§4.2의 4단계 설정이 명확합니다. "NVIC 미리 활성화" 팁이 불필요한 재설정을 방지합니다.

### 코드 길이
대부분 30줄 이하로 적절합니다. §4.5의 `uart_driver.c` 전체 코드가 약 45줄이지만, 이는 Driver 모듈 전체이므로 합리적입니다.

### 약간의 아쉬움
- §4.3의 블로킹 체험 코드에서 `last_led` 변수 선언이 누락되어 있습니다 (위에서도 언급).
- §4.4 main loop 코드의 LED 토글 부분에서 실제 `HAL_GPIO_TogglePin()` 호출이 빠져 있습니다 (line 561-564: `last_led = now;`만 있고 토글 코드 없음). 코드를 그대로 따라 하면 LED가 토글되지 않습니다.
