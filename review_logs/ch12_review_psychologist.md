## 1. 도입부의 감정 곡선 분석 (§12.0)

### 🟢 강점: "눈이 생긴다" - 완벽한 성취 프레이밍

**원고 텍스트**:
> "지금까지 우리는 숨겨진 기능들을 만들었습니다...
>  이제 '스마트 시계'는 정보를 **보여줘야** 합니다."

**심리 분석**:
1. **회상 (Recall)**: Ch10(모터 소리), Ch11(센서 데이터)을 회상하게 함
2. **한계 인식**: "하지만 정보는 보이지 않는다" → 공백감
3. **기대감 형성**: "이제 눈을 달아준다" → 해결책 제시
4. **자부심**: "비로소 시계 형태를 갖춘다" → 마일스톤 도달 감정

**감정 궤적**:
```
호기심(↑) → 불완전성 인식(↓) → 기대감(↑) → 자부심(↑)
```

**평가**: ★★★★★ (5/5 hearts) - 완벽함

### 🟡 개선 제안: 성취도 시각화
**현재**: 텍스트로만 설명
**제안**: 다음과 같은 진행 표시:

```html
<aside class="tip">
  💡 지금까지의 진행
  <br>
  Ch01-02: 기본 입출력 (GPIO, LED)
  <br>
  Ch03-09: 타이머, RTC, 모터 → 기능 완성 ✓
  <br>
  Ch10-11: 환경 센서 → 데이터 수집 ✓
  <br>
  <strong>Ch12: TFT LCD → 시각화 시작! 👀</strong>
  <br>
  (Ch13-16: 화면 화면 및 PC 대시보드)
</aside>
```

이렇게 하면 학생이 "나는 이미 이 정도를 했고, 이제 여기"라는 성취도를 더 명확히 느낍니다.

## 2. 복잡성 불안 관리 (§12.1 ~ §12.4)

### 🔴 Critical Issue: "25개 명령어"가 만드는 불안

**원고 텍스트** (§12.4, 라인 603):
> "ILI9341을 동작시키려면 25~35개의 초기화 명령어를 **정확한 순서대로** 전송해야 합니다."

**학생 심리 상태**:
1. **인지 과부하**: "25개? 그렇게 많아?"
2. **불안감**: "정확한 순서라니, 하나라도 빠뜨리면 안 되나?"
3. **자신감 하락**: "이게 정말 내가 할 수 있나?"

**실제 코드**: 15개만 구현

**심리적 영향**:
- ⚠️ 기대와 현실의 괴리로 인한 **혼동**
- ⚠️ "왜 25개가 아니라 15개일까?" 의문

### 🟠 Major Issue: 초기화 명령어 순서 강조가 과함

**문제 분석**:
```html
§12.4 제목: "ILI9341 초기화 명령어 시퀀스 ⭐ (핵심)"
라인 603: "25~35개의 초기화 명령어를 정확한 순서대로"
비유: "자동차 시동 절차"와 같습니다. 시동은 정해진 순서대로만..."
```

**심리 영향**:
- "반드시 순서를 지켜야 한다"는 강박감 형성
- "하나라도 틀리면 완전히 실패한다"는 이진 사고

### 🟢 긍정: 비유가 불안을 어느 정도 완화

**자동차 비유**:
> "시동은 정해진 순서대로만 진행됩니다..."

**심리 효과**:
- ✅ 불안이 아니라 "자연스러운 절차"로 재프레이밍
- ✅ "이미 알고 있는 예시"로 복잡성 축소
- ⚠️ 하지만 "왜 이 순서여야 하는가?" 설명 없음

### 🟡 권장: 불안 완화 메시지 추가

**제안** (§12.4 도입에 추가):
```html
<aside class="tip">
  💡 편하세요. 이 명령어들을 외울 필요는 없습니다.
  <br>
  "ILI9341 초기화는 LCD 제조사가 이미 최적화한 절차입니다.
  우리는 데이터시트의 코드를 복사해서 사용하면 됩니다.
  순서가 중요한 것은 맞지만, 우리는 이미 정확한 순서대로
  코드를 제공했으므로 안심하셔도 됩니다."
</aside>
```

**심리 효과**:
- ✅ "외울 필요 없다" → 인지 부하 감소
- ✅ "이미 정답이 준비됐다" → 불안감 감소
- ✅ "신뢰하면 된다" → 안전감 증대

## 5. 성취-불안-안심 곡선 (Emotional Arc)

### §12.0 도입부 분석

```
감정 상태

    기대감
      ↑
   자부심 ╱╲ (비로소 "시계"가 됨)
      ├─╱  ╲
      │    │ (이전 Ch10, 11의 불안감)
호기심 │   │  (왜 화면이 필요한가?)
      │  ╱   (화면 부재에 대한 공백)
      │╱
      └────────────────────
```

### 🟢 강점: Ch10 + Ch11과의 연결고리

**원고**:
> "Ch10에서는 시침을 움직였지만... Ch11에서는 온습도를 읽었지만...
>  이제 Ch12에서는 TFT LCD 화면을 초기화하여 모든 정보를 시각화합니다."

**심리 효과**:
- ✅ 이전 불안감(정보가 보이지 않는)을 이제 해결
- ✅ "이제야 문제를 풀 수 있다"는 해방감
- ✅ 누적된 성취의 **최종 결실**

### 🔴 문제: 초반 불안감 관리 부족

**§12.1 ~ §12.3의 기술적 설명**:
- SPI 프로토콜 (신호선 4개)
- ILI9341 핀맵 (Reset, D/C, CS)
- SPI DMA 아키텍처

이들은 "개념 설명"이 많아서, 학생이 "이것들이 정말 필요한가?" 의문 가능.

**개선 제안**: 각 섹션 시작에 "왜 이게 필요한가?" 설명 추가

```html
<!-- §12.1 시작에 추가 -->
<aside class="tip">
  💡 왜 SPI를 배워야 할까?
  <br>
  I2C는 느려서 LCD를 갱신할 수 없습니다.
  SPI는 I2C보다 50배 빠르므로, 초당 여러 번 화면을 그릴 수 있습니다.
  이번 섹션에서는 이 빠른 통신을 어떻게 하는지 배웁니다.
</aside>

<!-- §12.3 시작에 추가 -->
<aside class="tip">
  💡 왜 DMA를 써야 할까?
  <br>
  LCD의 초당 153,600 바이트를 모두 CPU가 전송하면 다른 일을 할 수 없습니다.
  DMA는 CPU를 깨우지 않고 메모리→LCD로 직접 데이터를 보냅니다.
  그래서 CPU는 다른 센서나 버튼을 동시에 처리할 수 있습니다.
</aside>
```
