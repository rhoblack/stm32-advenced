### 2.2 "편지의 제목과 내용" 비유 (D/C 신호)

**강의 관점 평가**: ⭐⭐⭐⭐ (4/5)

**강사 의견**:
```
좋은 비유이지만, 한 가지 놓친 부분이 있습니다.
편지의 "제목"은 **한 번** 쓰는 것이지만,
D/C 신호는 **각 그룹마다** 바뀝니다.
```

**개선된 비유**:
```
"D/C 신호는 우편배송 직원이 상자 겉에 붙이는 스티커입니다.
- 빨간 스티커: '명령어 포함'
- 파란 스티커: '데이터 포함'

각 상자마다 다른 색 스티커를 붙이므로,
LCD가 "어, 이건 명령어네" vs "이건 데이터네"를 구분합니다."
```

**강의 시 추가 팁**:
```
로직 애널라이저로 D/C 신호를 실시간으로 보여주면,
"아, 정말로 신호가 0→1→0→1로 바뀌는구나"라고 직관적 이해
```

### 2.3 "자동차 시동" 비유 (초기화 순서)

**강의 관점 평가**: ⭐⭐⭐ (3/5)

**평가**:
```
비유 자체는 좋지만, 실제 강의에서는 한계가 있습니다.
왜냐하면:
- 자동차: 순서가 고정적 (항상 점화→스타터→엔진)
- LCD: 일부 명령어는 순서를 바꿀 수 있음
```

**강사 추천 비유 개선**:
```
"초기화는 '신체의 점화 표현'과 같습니다.
- 심장을 먼저 깨워야 (Power Control)
- 폐를 활성화하고 (VCOMH)
- 눈을 열기 전에 (Display On)
- 신경계를 안정화시킵니다 (Gamma)

순서대로 하지 않으면 의식이 돌아오지 않습니다."
```

### 2.4 "직업 전화 vs 우편배송" 비유 (I2C vs SPI)

**강의 관점 평가**: ⭐⭐⭐ (3/5)

**문제점**:
```
"직업 전화"는 한국 학생에게 다소 낯선 표현입니다.
또한 왜 "일방향"이 빨라지는지 인과관계가 불명확합니다.
```

**강사 권장 비유**:
```
I2C: "택배 배송" - 느리지만 상품확인 (ACK) 가능, 여럿이 공유
SPI: "전화 통화" - 빠르고 직통, 한 명과만 통신 (CS로 선택)

시간 비유:
- I2C 100kbps → 초당 12,500 바이트 → 시계 한 번 갱신 30초
- SPI 10Mbps → 초당 1,250,000 바이트 → 시계 초당 8회 갱신
```

**강의 활동**:
```
1. 학생들에게 "우리가 초당 30회 갱신이 필요한 이유"를 물어보기
2. 답: "눈의 끝프레임(persistence of vision) 때문"
3. 그래서 SPI가 필요하다는 결론으로 자연스럽게 유도
```

```

### 🟡 개선: 디버깅 트리 추가

**현재**: 체크리스트는 있지만, "어떤 순서로 확인할까?" 가이드 부족

**강사 권장: 디버깅 플로우차트 추가** (§12.9에)

```
화면이 켜지지 않음
├─ 로그가 아무것도 안 나옴?
│  ├─ 예 → [1] 전원 확인
│  └─ 아니오 → [2] Reset 신호 확인

├─ "LCD 초기화 시작"까지 나오는가?

│  └─ 아니오 → Reset이 HIGH되지 않음
│
├─ "LCD 초기화 완료"까지 나오는가?
│  ├─ 예 → 색상 데이터 문제 ([7] 색상 설정)
│  └─ 아니오 → 명령어 순서 또는 D/C 문제 ([3] [4])

└─ 로그는 나오는데 화면이 안 켜짐?
   ├─ SPI 신호 확인 ([5] SPI 클럭, [8] CS 타이밍)
