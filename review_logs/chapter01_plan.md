## 4. 비유 아이디어 (기술 저자 제안)

| 개념 | 비유 | 한계 명시 |
|------|------|----------|
| Push-Pull vs Open-Drain | 수도꼭지(양방향) vs 배수구(단방향) — 배수구만 있으면 물을 채우려면 외부 수압(풀업)이 필요 | 전류 세기, 임피던스는 비유로 표현 안 됨 |
| 폴링 | 전자레인지 앞에 서서 "다 됐나?" 반복 확인 | 폴링에도 타임슬라이스 개념은 포함 안 됨 |
| 인터럽트 | 전자레인지 타이머 알람 — 다른 일 하다가 "띵!" 소리에 반응 | ISR 중첩·우선순위는 별도 설명 필요 |
| EXTI 엣지 트리거 | 엘리베이터 문 센서 — 열릴 때(상승), 닫힐 때(하강), 둘 다 감지 | 레벨 트리거와 비교 설명은 별도 필요 |
| NVIC PreemptPriority vs SubPriority | 응급실 환자 분류 — 응급도 등급(선점 가능) vs 같은 등급 내 순서(선점 불가) | 실제 ARM NVIC의 pending/active 상태까지는 비유 어려움 |
| 디바운싱 | 스프링 달린 문이 완전히 멈출 때까지 기다렸다가 판단하는 것 | 하드웨어 RC 필터 방식과 동작 원리 다름 |
| volatile 키워드 | 공유 화이트보드(메모리) vs 개인 메모지(CPU 레지스터 캐시) — volatile은 "매번 화이트보드에서 읽어라" | 멀티코어 동기화(memory barrier)와는 다름 |

## 7. 수강생 막힘 포인트 Top 5 (교육전문강사 분석)

1. **Open-Drain에서 "1이면 외부 풀업에 의존"한다는 개념** → 수도꼭지 비유 + 회로도 + I2C 연결
2. **EXTI 라인 번호와 GPIO 포트 번호의 매핑** (PA0, PB0 모두 EXTI0) → 제약 표 + 빨간 경고박스
3. **NVIC 우선순위 숫자가 낮을수록 높은 우선순위** → 응급실 계급 비유 + Priority Group 표
4. **"SW 디바운싱 vs HW 디바운싱 언제 선택?"** → 비용/납기 비교 표 + NUCLEO 내장 버튼 회로도 참조
5. **"폴링 vs 인터럽트, 내 프로젝트에서 뭘 써야 하나"** → 의사결정 플로우차트

## 10. NUCLEO-F411RE 특이사항 (강의 실습 주의)

| 이슈 | 내용 | 교재 처리 방법 |
|------|------|----------------|
| PC13 액티브 로우 | 버튼 누름 = LOW, 수강생이 "왜 0이 나오나?" 혼란 | 명시적 주석 + GPIO_PIN_RESET 조건문 |
| PA5 SPI1_SCK 핀 공유 | SPI 실습 시 충돌 가능, Ch01에서는 무관 | aside 박스로 미리 언급 |
| PC13 → EXTI15_10_IRQn | EXTI10~15 공유 핸들러, GPIO_PIN_13(비트마스크) 사용 필수 | `GPIO_PIN_13`과 리터럴 13 혼동 경고 |
| CN1 vs CN5 커넥터 | ST-LINK는 CN1, CN5는 전원만 | 강의 시작 전 케이블 포트 확인 체크리스트 |
| JP5 점퍼 | 전류 측정용 점퍼, 열리면 MCU 전원 차단 | 첫 실습 트러블슈팅 항목에 포함 |
