### Phase 1 합의 사항

**세 팀의 공통 강조 포인트:**
1. **SPI vs I2C 혼동 선제 해소** (교육강사 ★★★★★) → 도입부에 속도/핀/용도 비교 필수
2. **D/C 핀 타이밍이 핵심** (모두 일치) → 명령어 vs 데이터 신호 전이 충분한 시간 배분
3. **ILI9341 초기화 시퀀스 자세함** (기술 저자 강조) → 공식 초기화 코드 + 주석 상세
4. **DMA 전송 완료 콜백 + CS 타이밍** (기술 리뷰어 예상 관심) → 라이브 타이밍 다이어그램 필수
5. **Ch11 I2C 완성 → Ch12 SPI 대비** (교육설계자) → 유사 아키텍처(Driver 레이어) + 차이점(속도/핀) 명확화
6. **감정 곡선**: 성취감(시각 정보!) → 호기심(컬러 LCD?) → 불안(초기화 명령어 많음) → 이해(패턴) → 성취(픽셀 그리기) → 자부심(화면 완성)

**기획안 차이 조정:**
- **절 구조**: 기술 저자(8절) vs 교육설계자(7절) vs 강사(9절) → **9절 채택** (도입부 SPI 개념 / D/C 타이밍 / 초기화 / 데이터 전송 / 픽셀/도형/텍스트 / GFX 기초 / 정리)
- **신개념 압축**: 5개 → 4개 (SPI 통신 규약 / ILI9341 명령어 형식 / D/C 제어 신호 / DMA 기반 전송)
- **시간 배분**: 이론 40% + 실습 56% + 휴식 4% 채택
- **하드웨어 호환성**: ILI9341 SPI 42MHz 이하 권장 (NUCLEO-F411RE STM32F411: APB1 = 45MHz → PSC=2 또는 더 높이 필요)

### §12.1 SPI 통신 프로토콜 기초 (40분)
**학습 목표**: [이해] SPI 신호(SCLK, MOSI, MISO, CS) + 타이밍
**핵심 개념** (1개):
- **SPI(Serial Peripheral Interface)** = "4선 고속 동기 통신" (I2C 2선 대비)
  - SCLK: 클럭 (마스터 생성, 최대 42MHz 추천)
  - MOSI: Master Out Slave In (MCU → LCD)
  - MISO: Master In Slave Out (LCD → MCU, 보통 사용 안 함)
  - CS/NSS: Chip Select (활성 LOW)

**비유 및 시연**:
- **I2C 비유**: "우편물 배송 (느림, 확인 신호 있음)"
- **SPI 비유**: "직업 전화 (빠름, 일방향 고속 전송)"
- **CPOL/CPHA 설명**: 클럭 극성 / 샘플 타이밍 (STM32F411 기본값: CPOL=0, CPHA=0)

**아키텍처 배치**:
- v1.1 (I2C Driver + SHT31) → v1.2 (SPI Driver + LCD Driver 추가)
- 도입부에서 전체 아키텍처: HAL → Driver → Device Driver → Service

### §12.2 ILI9341 디스플레이 하드웨어 이해 (35분)
**학습 목표**: [이해] ILI9341 핀 배치 / Reset 시퀀스 / 초기화 명령어 형식
**핵심 개념** (2개):
1. **ILI9341 핀**: SPI(SCL/SDA) + D/C(Command/Data) + CS + RESET
2. **초기화 시퀀스**: RESET(LOW→HIGH) + 초기화 명령어 시퀀스

**하드웨어 연결**:
- SPI1: PB3(SCLK), PB5(MOSI), PB4(MISO)
- **D/C 핀(Command/Data)**: 수동 GPIO (예: PA4)
  - D/C=0: 다음 바이트는 명령어 (Command)
  - D/C=1: 다음 바이트는 데이터 (Data)
- **CS 핀(Chip Select)**: GPIO 수동 제어 (예: PA3)
- **RESET 핀**: GPIO 또는 전원 직접 (일반적으로 고정 HIGH)

**비유**:
- "D/C 핀 = 편지의 '제목' vs '내용'" → 먼저 제목(명령어)을 전하고, 그 다음 내용(데이터)을 전합니다
- "CS 핀 = 전화 연결" → CS=LOW일 때만 통신, 끝나면 CS=HIGH

**실무 주의**:
- ILI9341은 **SPI Mode 0** (CPOL=0, CPHA=0)
- 클럭 속도: 최대 42MHz (일반적으로 10~20MHz 권장)
- 초기화는 "회사 SOP" 같음 → 순서와 타이밍 중요

### §12.3 SPI DMA 초기화 및 구조 (50분)
**학습 목표**: [적용] STM32CubeIDE SPI1 DMA 설정 + HAL API 이해
**핵심 개념** (1개):
- **SPI DMA 초기화**: TX-only DMA (보통 MISO는 사용 안 함)
- DMA 완료 콜백을 이용한 CS 타이밍 제어

**CubeMX 설정**:
1. SPI1 Master Mode, CPOL=0, CPHA=0, Clock 10MHz (LSB first or MSB first는 고려)
2. DMA: Stream 2 Channel 0 (SPI1 TX)
3. DMA Mode: **NOT Circular** (1회성 전송)
4. DMA 완료 인터럽트 활성화

**실습 흐름**:
1. SPI1 기본 설정 + DMA TX
2. GPIO PA3(CS), PA4(D/C) 설정
3. LCD Reset 시퀀스 (RESET GPIO 토글 또는 전원 초기 대기)
4. 초기화 명령어 전송: `HAL_SPI_Transmit_DMA(...)`

**비유**:
- "DMA 없이 SPI 송신" = "우체부가 편지를 한 바이트씩 전달" (느림)
- "DMA 사용" = "우체부가 봉투 전체를 한 번에 배달" (빠름)

**실무 주의**:
- **DMA 완료 콜백에서 CS=HIGH 설정** (전송 완료 후 선택 해제)
- DMA 버퍼는 **static** 또는 heap에 할당 (스택 사용 금지)

### §12.4 ILI9341 초기화 명령어 시퀀스 (60분) ⭐
**학습 목표**: [적용/분석] 공식 ILI9341 초기화 코드 이해 및 구현
**핵심 개념** (1개):
- **ILI9341 초기화 명령어 시퀀스** (25~35개 명령어)

**초기화 패턴** (반복):
```
// 명령어 전송
D/C = 0
SPI_Transmit_DMA(command_byte)   // 0xEF, 0xEB, ... 등
delay(짧은 시간, ~5ms)

// 데이터 전송
D/C = 1
SPI_Transmit_DMA(data_bytes)     // 0x03, 0x80, ... 등
delay(짧은 시간)

// 반복...
```

**주요 초기화 명령어** (10가지):
1. **0xEF** (Power Control A) → 데이터 3바이트
2. **0xCB** (Power Control B) → 데이터 5바이트
3. **0xC0** (Power Control 1) → 데이터 1바이트
4. **0xC1** (Power Control 2) → 데이터 1바이트
5. **0xC5** (VCOMH / VCMOL 설정) → 데이터 2바이트
6. **0xC7** (VCOMH 전압 조정) → 데이터 1바이트
7. **0x36** (Memory Access Control) → 데이터 1바이트 (색상 순서: RGB/BGR, 방향)
8. **0x3A** (Pixel Format Set) → 데이터 1바이트 (16-bit: 0x55)
9. **0xB1** (Frame Rate Control) → 데이터 2바이트
10. **0xB6** (Display Function Control) → 데이터 3바이트
11. **0x11** (Sleep Out) → 데이터 없음
12. **0x29** (Display On) → 데이터 없음

**비유**:
- "ILI9341 초기화" = "자동차 시동 절차" → 순서대로 수행, 하나라도 빠지면 시작 안 됨

**실무 전략**:
- 공식 데이터시트 초기화 코드를 **그대로 복사** (레지스터 값은 변경 금지)
- 각 명령어 사이에 **짧은 딜레이** (1~5ms) 필수
- `hal_delay()` 또는 `HAL_Delay()`는 마이크로초 정밀도 부족 → `volatile int i; for(i=0; i<10000; i++);` 권장 (또는 TIM 사용)

**교육강사 주의**:
- "이 명령어들이 뭔지 하나하나 외울 필요 없습니다. 하지만 **순서**와 **타이밍**은 중요합니다."

### §12.5 LCD 기본 색상 설정 및 픽셀 쓰기 (50분)
**학습 목표**: [적용] 색상 설정 (16-bit RGB565) + 개별 픽셀 그리기
**핵심 개념** (1개):
- **RGB565 색상**: R(5비트) G(6비트) B(5비트) = 65,536가지 색상

**색상 포맷**:
```
RRRRR GGGGGG BBBBB (16비트)

RGB565(red=255, green=128, blue=0)
= (255 << 11) | (128 << 5) | 0
= (31 << 11) | (32 << 5) | 0  // 정규화
= 0xFC00
```

**픽셀 쓰기 절차**:
1. **0x2A** (Set Column Address) + 시작 X, 끝 X (2바이트 × 2)
2. **0x2B** (Set Row Address) + 시작 Y, 끝 Y (2바이트 × 2)
3. **0x2C** (Write Memory) + 색상 데이터 (2바이트 per 픽셀)

**비유**:
- "픽셀 쓰기" = "그림판에서 사각형 선택 후 색칠하기" → 먼저 범위(Column/Row) → 그 다음 색상 데이터

**실습**:
1. 전체 화면 빨강(0xF800) 채우기 (320×240 × 2바이트 = 153,600바이트)
2. 중앙에 초록(0x07E0) 사각형 그리기 (100×100)
3. 왼쪽 위에 파랑(0x001F) 점 4개 그리기

**실무 팁**:
- 전체 화면은 DMA 버퍼 크기 제한 확인 필수 (SRAM 부족)
- 부분 화면 업데이트 > 전체 화면 (성능)

## 핵심 메시지 3가지

### 1️⃣ "왜 화면인가?" — 프로젝트 완성으로의 길
> "Ch10에서 시침을 움직이고, Ch11에서 온습도를 읽었습니다. 하지만 사용자는 **보고 싶어 합니다**. 이제 그 모든 데이터를 스마트 시계 화면에 표시합니다. 이것이 '스마트 시계'의 진정한 시작입니다."

### 2️⃣ "SPI는 왜 빠른가?" — 속도의 중요성
> "I2C는 한 바이트 전송마다 ACK를 기다려야 합니다(느림). SPI는 SCLK만 맞추고 계속 보냅니다(빠름). LCD처럼 **대용량 데이터**를 빠르게 보내야 할 때는 SPI를 선택합니다. DMA까지 활용하면 **CPU 간섭 없이** 화면을 갱신할 수 있습니다."

### 3️⃣ "D/C 핀이 핵심" — 명령어와 데이터의 경계
> "ILI9341은 같은 SPI 신호로 명령어와 데이터를 모두 전송합니다. D/C=0이면 '이제 전송하는 것은 명령어'라는 신호, D/C=1이면 '이제 전송하는 것은 데이터'라는 신호입니다. 초기화 시퀀스는 이 명령어와 데이터를 **정확한 순서로** 보내는 과정입니다."
