# Ch14. I2C DMA 심화 — 독자 리뷰 (Beginner Reader Review)
작성자: 초보자 독자 에이전트 (중급 임베디드 개발자 관점)
작성일: 2026-03-17

## 독자 프로필 확인
- C언어 숙련, HAL 기본 사용 경험 있음
- DMA는 Ch05에서 기초 학습, UART DMA는 Ch06에서 경험
- I2C는 Ch11에서 폴링 방식으로 경험
- SHT31 드라이버는 이미 작성해 본 상태

---

## 전체 이해도 평가

**종합 이해도**: ⭐⭐⭐⭐ (4/5)

이 챕터는 이전 챕터(Ch05 DMA 기초, Ch11 I2C 폴링) 지식을 잘 연결하여 설명하고 있습니다.
특히 "커피숍 직원" 비유와 CPU 타임라인 SVG가 DMA 전환의 동기를 매우 잘 설명합니다.
콜백 체인 패턴이 처음에는 생소하지만, 코드 단계별 분해가 잘 되어 있어 따라갈 수 있습니다.

---

## 절별 이해도 평가

### 1절: I2C DMA 전환 이유 — ⭐⭐⭐⭐⭐ (5/5)

CPU 타임라인 SVG와 4.2% vs 0.2% 수치가 직관적으로 이해됩니다.
"왜 DMA가 필요한가"에 대한 답이 명확합니다.
DMA 전환의 손익 계산 팁도 실용적입니다.

### 2절: HAL I2C DMA API — ⭐⭐⭐⭐ (4/5)

폴링 vs DMA 함수 시그니처 비교가 명확합니다.
CubeMX 설정 7단계 절차가 구체적이어서 따라하기 쉽습니다.

**이해 어려운 부분**:
"ISR 콜백에서 float 연산을 피하라"는 팁에서 왜 그래야 하는지 설명이 부족합니다.
"FPU lazy stacking"이라는 용어는 처음 접하는 독자에게 생소합니다.
"ISR에서는 빠른 처리만 하세요"라는 실용적 규칙으로 단순화하면 충분합니다.

### 3절: Repeated Start 미지원 — ⭐⭐⭐⭐ (4/5)

"택시 환승 vs 새 택시 잡기" 비유가 직관적입니다.
SVG가 두 방식의 차이를 명확히 보여줍니다.

**이해 어려운 부분**:
"SHT31이 Stop+Start를 허용한다"는 사실이 왜인지 설명이 없습니다.
단순히 "SHT31 데이터시트에서 Repeated Start가 필수가 아님을 확인했다"는 한 문장만 추가해도 독자의 신뢰도가 높아집니다.

콜백 체인 패턴에서 on_tx_cplt → TIM6 → on_rx_cplt 흐름은 처음 보면 "왜 타이머를 쓰는가?"라는 질문이 자연스럽게 나옵니다. HAL_BUSY 문제와 SHT31 대기 시간을 동시에 해결한다는 점을 더 강조하면 좋겠습니다.

### 4절: 다중 DMA 동시 운용 — ⭐⭐⭐⭐ (4/5)

"고속도로 차선" 비유가 스트림 충돌을 이해하는 데 효과적입니다.
DMA 스트림 배치표 SVG가 눈으로 보기 좋습니다.

**이해 어려운 부분**:
DMA 스트림 우선순위(DMA_PRIORITY_VERY_HIGH 등)와 NVIC 우선순위(PreemptPriority)의 차이가 처음에는 헷갈립니다. 두 개념을 구분하는 간단한 표가 있으면 좋겠습니다.

### 5절: 연속 측정 루프 — ⭐⭐⭐⭐ (4/5)

메인 루프 통합 예제가 전체 그림을 보여줘서 좋습니다.
정상 로그 출력 예시가 "내가 제대로 구현하면 이런 결과가 나온다"는 기대를 만들어 줍니다.

**이해 어려운 부분**:
"재진입 문제"(Re-entrant)라는 용어가 처음 등장하는데 간단한 설명이 있으면 좋겠습니다.
"함수가 실행 중에 다시 호출되는 것"이라는 한 문장 설명이 있으면 충분합니다.

---

## 설명 없이 등장하는 용어 발견

| 용어 | 위치 | 처리 현황 |
|------|------|----------|
| FPU lazy stacking | 2절 팁 | ❌ 설명 없음 → 제거하거나 1줄 설명 추가 필요 |
| ISR (Interrupt Service Routine) | 여러 곳 | ⚠ Ch05에서 다뤘으나 간략 리마인드 권장 |
| 재진입(Re-entrant) | 5절 팁 | ❌ 설명 없음 → 1줄 설명 필요 |

| 연결 | 평가 |
|------|------|
| Ch05 DMA 기초 → Ch14 I2C DMA | ✅ 자연스럽게 연결됨 |

| Ch11 SHT31 CRC-8 → 재사용 | ✅ "Ch11 로직 재사용" 명시됨 |

## 수정 제안 (우선순위)

1. **[권장]** 2절 팁에서 "FPU lazy stacking" 대신 "ISR에서는 최대한 빠른 처리만 하고, 무거운 연산은 메인 루프에서" 로 단순화
2. **[권장]** 5절 팁에서 "재진입(Re-entrant)"에 괄호 설명 추가: "재진입 문제 — 함수가 실행 완료 전에 다시 호출되는 상황"
3. **[선택]** 4절에 DMA 하드웨어 우선순위 vs NVIC 우선순위 구분 표 추가
