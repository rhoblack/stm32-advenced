## 개선 요청 사항

### 어려운 부분 1 (4절 — HFSM 부모 exit/entry 순서)

**문제**: "자식 exit → 부모 exit → 부모 entry → 자식 entry" 순서가 텍스트로만 설명되어 있음.

**독자 관점 피드백**:
처음 읽으면 "STOPWATCH_VIEW → IDLE 전이 시 정확히 어떤 순서로 함수가 호출되는가?"가
한 번에 들어오지 않습니다. FAQ에서 설명하고 있지만, 이 순서를 보여주는 작은 콜 스택 예시나
SVG의 진입/진출 화살표 순서 표시가 있으면 더 좋겠습니다.

**제안**: FAQ 아래에 콜 스택 형식의 4단계 예시 추가
```
STOPWATCH_VIEW → IDLE 전이 콜 스택:
1. hfsm_dispatch(TIMEOUT)
2. handle_stopwatch_view() → false (위임)
3. handle_active(TIMEOUT) → true, next=IDLE
4. exit_stopwatch_view()  ← 자식 exit
5. exit_active()          ← 부모 exit
6. entry_idle()           ← 다음 상태 entry
```

### 어려운 부분 2 (3절 — guard 함수 여러 개 조합)

**문제**: "여러 guard 함수를 AND/OR 조합하려면 guard 함수 내에서 조합하거나 테이블에 행을 여러 개 추가합니다"라는 설명이 추상적.

**제안**: 구체적 예시 코드 한 줄 추가
```c
/* guard 함수 내 조합 예시 */
static bool guard_alarm_and_display_on(void) {
    return alarm_service_is_triggered() && ui_is_display_on();
}
```

### 용어 설명 누락 (Minor)

- "OCP(개방 폐쇄 원칙)"이 3절 코드 주석에 등장하지만 본문에 설명 없음. 처음 등장 시 "(Open-Closed Principle — 확장에는 열려 있고 수정에는 닫혀 있어야 한다)" 설명 병기 필요.
