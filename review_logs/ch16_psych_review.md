## 3. 감정 곡선 분석

```
1절 (프로토콜 설계)
  → 호기심: "바이너리로 전환하면 뭐가 좋지?"
  ↓
2절 (프레임 + CRC)
  → 약간의 불안: "CRC 코드가 무섭게 생겼다"
  → 안심: "블랙박스로 써도 된다" / "영수증 비유"
  ↓
3절 (MCU 구현)
  → 이해 + 집중: "build_frame() 6단계, 따라갈 수 있겠다"
  → 소성취: "내 코드에서 프레임이 만들어졌다"
  ↓
4절 (Python 분석)
  → 가벼운 호기심: "스레드가 어떻게 생겼지?" (과제 없음)
  ↓
5절 (대시보드 실행)
  → 흥분 + 성취: "Python 화면에 내 MCU 데이터가 뜬다!"
  ↓
6절 (전체 아키텍처)
  → 성취감 절정: "16개 챕터를 지나 여기까지 왔다"
```

**평가**: ⭐⭐⭐⭐⭐ — 설계된 감정 곡선이 실현됩니다.
