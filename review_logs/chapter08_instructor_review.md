## 8. 코드 버그 발견 (필수 수정)

### BUG: §8.7 main.c App 레이어 코드 — BTN_SHORT가 BTN_LONG으로 잘못 기재

**위치**: manuscripts/part2/chapter08.html line 1048~1052

```c
if (duration >= 2000) {
    stopwatch_handle_event(SW_EVT_BTN_LONG);
} else {
    stopwatch_handle_event(SW_EVT_BTN_LONG);  // BUG! SW_EVT_BTN_SHORT이어야 함
}
```

**문제**: else 분기(짧게 누름)에서도 `SW_EVT_BTN_LONG`을 전송. 결과적으로 모든 버튼 입력이 LONG으로 처리되어 Start/Pause/Resume이 동작하지 않고 Reset만 됨.

**수정**: else 분기를 `SW_EVT_BTN_SHORT`로 변경 필수.

**심각도**: **CRITICAL** — 이 코드를 그대로 실습하면 스톱워치가 전혀 동작하지 않음. 학생들이 "코드 그대로 쳤는데 안 돼요"라는 상황 발생.

### BUG: 버튼 길게 누르기 로직 결함

**위치**: 같은 코드 블록 (line 1044~1053)

```c
if (g_btn_pressed) {
    g_btn_pressed = 0;
    uint32_t duration = HAL_GetTick() - g_btn_press_ms;
```

**문제**: `g_btn_press_ms`는 EXTI 콜백(line 1025)에서 버튼 **눌린 순간** 기록. 그러나 `duration` 계산은 **메인 루프 폴링 시점**에서 수행. EXTI는 rising/falling edge 한 번에 플래그를 세트하므로, 이 로직으로는 "길게 누르기"를 정확히 감지할 수 없음.

**권장 해결**: 버튼 release 감지(falling edge) 또는 별도 타이머 기반 long-press 감지 로직이 필요. 다만 이 코드가 "개념 설명용 스켈레톤"이라면, 주석으로 "실제 구현에서는 release 이벤트 감지 필요" 명시 권장.

**심각도**: MEDIUM — 개념 전달에는 지장 없으나, 그대로 실행 시 long press 미동작.
