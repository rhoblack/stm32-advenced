## 2. 섹션별 이해도 분석

### 섹션 1: 28BYJ-48 스트라이드

⭐⭐⭐⭐ (4/5)

**이해하기 쉬운 점**:
- 계단 비유 (line 134-139) 명확하고 직관적 ✓
- 5.625° = 360÷64 공식 쉬움 ✓
- 기어비 64:1 설명 있음 ✓

**약간 어려운 점**:
🟡 **N1**: 기어비 설명이 조금 추상적
```
line 147-171:
"기본 1.8°에 내부 기어 감속 64:1이 적용되어 있습니다:
  1.8° × (1 ÷ 64) × 16 = 5.625°"
```

- "1 ÷ 64"는 맞지만, "× 16"이 갑자기 나옴
- 정확히는: 1.8° × (1/4) = 0.45° (기본 극 4개), 그 다음 64:1 감속
- 최종: (1.8° ÷ 64 × 16 스텝) 하지만 설명이 순서대로 아님

**권장 개선**:
```
더 단계적으로:
Step 1: 기본 스텝 = 1.8°
Step 2: 4개 코일이므로 4 극 = 1.8° × 4 = 7.2° (풀 회전)
Step 3: 기어 감속 64:1 → 7.2° / 64 ≈ 0.11° (하지만 이것도 아님)

실제:
Step 1: 스텝 모터는 다단계 코일 제어로 여러 스텝으로 나뉨
Step 2: 28BYJ-48은 실제로 8개 극을 가진 모터
Step 3: 1.8° × (1/8) 스텝 = 0.225°
Step 4: 64:1 기어감속 → 0.225° × 25 = 5.625°
```

**현재 수준으로 충분하나**, 더 구체적인 단계별 설명이 있으면 좋음

### 섹션 3: stepper_driver.c 구현

⭐⭐⭐⭐ (4/5)

**이해도**:
- stepper_set_coils() 함수 로직 명확
- stepper_step_forward/backward 반복 구조 쉬움
- active-low 반전 로직 설명 있음 (line 377-379) ✓

**약간 어려운 점**:
🟡 **N2**: 여자 시퀀스 배열이 왜 이 순서인지 설명 부족

```c
static const uint8_t full_step_sequence[4][4] = {
    {1, 0, 0, 0},  /* Step 0: A만 활성 */
    {1, 1, 0, 0},  /* Step 1: A, B 활성 */
    {0, 1, 0, 0},  /* Step 2: B만 활성 */
    {0, 1, 1, 0},  /* Step 3: B, C 활성 */
};
```

- 왜 A → A+B → B → B+C 순서인가?
- "시계방향 회전을 위해" 정도만 있음
- 반시계방향이면 역순이어야 한다는 설명 없음 (현재 line 420-428 반시계 구현)

**현재 코드**:
```c
motor_step_index = (motor_step_index + 3) % 4;  /* -1과 동일 */
```
- 이 트릭 자체는 우수하지만, 왜 "+3 % 4 = -1"인지 설명 추가 권장

🟡 **N3**: stepper_home() 함수 이해
```c
void stepper_home(void)
{
    LOG_I("Homing motor to 0 degrees");
    stepper_move_steps(-motor_position, 30);  /* 현재 위치에서 역방향 이동 */
    motor_position = 0;
    motor_step_index = 0;
}
```
- 논리는 맞지만, "home" 개념이 아직 명확하지 않음
- 시계의 시침 0° = 12시 위치 이해 필요 (이전 챕터에서?)
