## 2. Critical/Major 이슈 목록 (중복 제거 후)

### [M-1] log_print() 버퍼 오버플로 잠재 위험 (기술 Major-1)
- **내용:** `snprintf` offset 누적 후 `vsnprintf` 호출 시 버퍼 경계 초과 가능
- **수정:** offset을 `sizeof(buf) - 3` 이하로 클램핑하는 방어 코드 추가
- **우선순위:** 정확성 → **반영**

### [M-2] ITM_SendChar() busy-wait 설명 오류 (기술 Major-2 = 강사 Major-2)
- **내용:** "ISR 내 LOG_D 사용 금지" 이유로 "항상 busy-wait" 설명은 부정확
- **정확한 사실:** ITM 비활성화(디버거 미연결) 상태에서는 즉시 반환됨. busy-wait는 ITM 활성화 + FIFO 가득 찬 경우만 발생
- **수정:** 주의사항 박스 설명 수정
- **우선순위:** 정확성 → **반영**

### [M-3] 실습 예상 로그 출력 함수명 컬럼 불일치 (기술 Minor-5 = 강사 Major-3)
- **내용:** `%-12s`는 최소 너비(패딩)이지 최대 너비(잘림)가 아님. `HAL_GPIO_EXTI_Callback`(22자)이 잘리지 않고 전부 출력됨
- **수정:** 예상 출력 예시 수정 또는 `%.12s` (최대 너비 절단)로 포맷 변경
- **우선순위:** 정확성 → **반영**

### [M-4] RTT 라이브러리 설치 절차 누락 (독자 Major-1 = 강사 Major-1 = 심리 Major-1)
- **내용:** §2.2에서 SEGGER_RTT_printf() 코드가 나오지만 라이브러리 파일 취득 및 CubeIDE 추가 방법이 없음
- **수정:** SEGGER RTT 라이브러리 다운로드 위치 및 CubeIDE 프로젝트 추가 단계 간략 서술 추가
- **우선순위:** 이해도 → **반영**

### [M-5] setvbuf() 미사용 시 문제 미설명 (독자 Major-2)
- **내용:** `setvbuf(stdout, NULL, _IONBF, 0)` 코드가 있지만 왜 필요한지 설명 없음
- **수정:** "printf 버퍼링으로 마지막 로그가 유실될 수 있다" 간략 설명 추가
- **우선순위:** 이해도 → **반영**

### [M-6] §2.4 ##__VA_ARGS__ 설명 불안 선점 문제 (심리 Major-2)
- **내용:** "##없으면 컴파일 에러" 설명이 학습 시작 전 불안 유발. GCC에서는 실제 에러 없음
- **수정:** "GCC 환경(CubeIDE)에서는 완전히 지원됨"을 먼저 서술, 표준 제약은 배경 지식으로 후술
- **우선순위:** 심리적 안전 → **반영**

### [M-7] §2.3 오타 수정 (독자 Minor)
- **내용:** "DEBUG **레그**를" → "DEBUG **레벨**을"
- **수정:** 오타 수정
- **우선순위:** 정확성 → **반영**
