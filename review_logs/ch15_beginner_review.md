## 어려운 부분 및 개선 제안

### 이슈 #1: 2절 — strtok_r 반환 후 `argv[]`가 어떻게 채워지는지 불명확

**문제**: SVG 다이어그램이 각 토큰을 색상으로 구분하지만,
`argv[0] = 문자열 "time"`이 아니라 `line_buf` 내의 주소라는 점이 명확하지 않습니다.
C 포인터에 익숙한 독자는 자명하지만, 다소 경험이 부족한 중급 개발자는
"argv[0]이 원본 버퍼를 가리킨다"는 사실을 모를 수 있습니다.

**제안**: SVG 다이어그램에 `argv[0]` → `&line_buf[0]` 화살표를 추가하거나,
짧은 주석 "argv 배열은 line_buf 안의 토큰 시작 주소를 저장합니다 (복사가 아님)"을 추가합니다.

### 이슈 #2: 3절 — `s_cmd_table`이 `static const`인 이유 설명 없음

**문제**: `static const cmd_entry_t s_cmd_table[]`에서 `static const`의 의미를 모르는 독자가 있습니다.
특히 임베디드에서 `const` 배열이 Flash에 저장된다는 점을 처음 접하는 경우입니다.

**제안**: 짧은 aside로 설명 추가:
> `static`은 파일 범위 제한, `const`는 수정 불가 → 컴파일러가 Flash(ROM)에 배치합니다.
> RAM이 부족한 임베디드에서 룩업 테이블을 `const`로 선언하는 것이 중요한 이유입니다.

### 이슈 #3: 실습 — ring_buf_t의 `g_uart_rx_buf` 선언 위치를 독자가 찾아야 함

**문제**: `extern ring_buf_t g_uart_rx_buf;`가 Ch06에서 선언되었다고 언급하지만,
정확히 어느 파일에 있는지 명시하지 않습니다.

**제안**: 주석으로 파일 경로 명시: `/* uart_driver.c 또는 ring_buf.c 에 선언 */`
