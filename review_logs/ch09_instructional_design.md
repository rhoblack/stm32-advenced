## 2. Ch08과의 누적 성장 분석

### 2.1 선행 지식 의존성 맵

| 선행 챕터 | 활용 개념 | Ch09에서의 역할 |
|-----------|-----------|----------------|
| Ch01 (GPIO/EXTI) | EXTI 인터럽트 | 알람 트리거 시 LED/부저 동작 |
| Ch02 (로그) | LOG_I/LOG_D | RTC 시각 출력, 디버깅 |
| Ch03 (아키텍처) | 4계층 레이어, FSM 기초 | Driver→Service 패턴, ALARM FSM 설계 |
| Ch07 (TIM 기초) | PSC/ARR 주기 계산 | RTC prescaler 개념 유사성 (PREDIV_A × PREDIV_S) |
| Ch08 (Advanced TIM) | Service 레이어, FSM 테이블 | stopwatch_service → alarm_service 패턴 반복 |

### 2.2 v0.8 → v0.9 성장 포인트

```
v0.8 (Ch08)                          v0.9 (Ch09)
──────────────────                   ──────────────────
타이머 기반 "경과 시간"        →     RTC 기반 "실제 시각"
stopwatch_service (FSM)        →     alarm_service (FSM) — 패턴 반복
TIM 카운터 (상대 시간)         →     RTC BCD (절대 시간, 년월일시분초)
Service 레이어 1개             →     Service 레이어 2개 (누적)
tim_driver 확장                →     rtc_driver 신규 (Driver 패턴 재사용)
```

### 2.3 핵심 성장 메시지
- **"상대 시간 → 절대 시간"**: Ch07-08의 TIM은 "얼마나 지났나"(상대), Ch09의 RTC는 "지금 몇 시인가"(절대)
- **Service 레이어 패턴 반복**: Ch08 stopwatch_service 구조를 alarm_service에 그대로 적용 → 패턴 체화
- **FSM 설계 반복**: STOPWATCH FSM(3상태) → ALARM FSM(4상태) — 상태 1개 추가로 난이도 점진적 증가
- **Ch10 예고**: RTC 시각 → 스텝모터 시침 각도 변환 (v1.0 마일스톤)

### 인지 부하 관리 전략

1. **§9.1 BCD 난관 해소**:
   - 비유: "디지털 시계 디스플레이" — 각 자릿수가 독립적으로 0~9 표시 (BCD = 자릿수별 독립 인코딩)
   - 비교 표: BCD vs Binary 변환 예시 (23시 = 0x23 BCD = 35 decimal)
   - 즉시 실습: BCD 변환 코드 작성 → 성공 체험

2. **§9.4 알람 마스크 해소**:
   - 비유: "알람 시계의 요일 반복 설정" — 마스크 = "이 자릿수는 무시해" (매일 7시 = 날짜 마스크)
   - 진리표: 마스크 비트 조합 → 알람 동작 예시

3. **§9.6 FSM 인지 부하 최소화**:
   - Ch08 STOPWATCH FSM 구조를 먼저 복습 (2문장)
   - ALARM FSM은 STOPWATCH와 동일 패턴임을 강조
   - 차이점만 집중: 상태 1개 추가(ACKNOWLEDGED), 이벤트 이름 변경

## 9. NUCLEO-F411RE 하드웨어 주의사항

### LSE 크리스탈 상태
- **기본**: LSE 미장착 (SB48=OFF, SB49=OFF)
- **LSE 사용 시**: SB48, SB49 솔더 브릿지 장착 + 32.768kHz 크리스탈 필요
- **교육 환경 권장**: LSI 사용 (추가 하드웨어 불필요)
- **정확도 트레이드오프**: LSI ±5% → 하루 최대 ~72분 오차 (실습용으로는 충분)

### 실습 권장 구성
```
RTC 클럭: LSI (32kHz 내부 RC)
알람 출력: LED (PA5, 온보드 LD2)
알람 확인: 버튼 (PC13, 온보드 B1)
로그: UART2 (PA2/PA3, ST-LINK 가상 COM)
```
