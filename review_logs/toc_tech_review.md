## 🔴 Critical 이슈 (기술적 오류 또는 심각한 순서 문제)

### C-1. Ch06 Advanced TIM 심화 — Basic TIM / General Purpose TIM 선수 챕터 누락
- **문제**: Ch06이 "Advanced TIM 심화"로 시작하나, 타이머 기초(TIM 구조, PSC/ARR/CCR 레지스터, UEV 이벤트, HAL_TIM_Base_Start_IT) 챕터가 전혀 존재하지 않는다.
- **영향**: 중급 개발자라도 STM32 타이머는 TIM2~TIM5(General Purpose)와 TIM1(Advanced Control)의 차이, Prescaler·Auto-Reload 계산 방식을 먼저 다룬 후 PWM Complementary·Dead-time으로 진입해야 한다.
- **구체적 기술 사항**: NUCLEO-F411RE에서 TIM1은 Advanced Control Timer(Break, Dead-time, Complementary), TIM2/3/4/5는 General Purpose, TIM9/10/11은 간이 타이머로 구분된다. 이 구분 없이 Complementary 출력과 Dead-time부터 설명하면 학습자 혼란이 크다.
- **권고**: Ch06 이전에 "타이머 기초 (General Purpose TIM, PWM 기본)" 챕터를 삽입하거나, Ch06 도입부에 최소 30분 분량의 TIM 구조 설명을 명시적으로 포함시킬 것.

### C-2. Ch10 SPI DMA — 기초 SPI 챕터 누락으로 논리적 도약 과다
- **문제**: SPI 폴링(HAL_SPI_Transmit / HAL_SPI_Receive) 및 SPI 인터럽트를 다루는 챕터 없이 곧바로 SPI DMA + ILI9341 LCD 드라이버 설계로 진입한다.
- **영향**: ILI9341은 D/C 핀 제어, CS 핀 수동 관리(HAL_GPIO_WritePin), 8/16비트 컬러 패킹 등 SPI 외에도 복잡한 HW 인터페이스 요소가 있다. SPI 프로토콜(CPOL/CPHA, 클럭 속도, NSS 처리)을 모르는 상태에서 DMA 레이어까지 한 챕터에 압축하면 기술 완결성이 낮아진다.
- **권고**: Ch09-Ch10 사이 또는 Ch10 내에 "SPI 기초 (폴링 → 인터럽트)" 절을 추가하거나, Ch04 DMA 챕터처럼 SPI 전용 기초 챕터를 Part 3 앞에 배치할 것.

### C-3. Ch02 링 버퍼 기반 비동기 로그 — UART 챕터(Ch03) 이전 배치
- **문제**: Ch02에서 "링 버퍼 기반 비동기 로그 출력"을 다루는데, UART 자체는 Ch03에서 처음 소개된다. 링 버퍼 기반 비동기 출력은 UART TX 메커니즘을 전제로 해야 한다.
- **영향**: Ch02에서 링 버퍼 설계를 구현하더라도 실제로 UART로 내보내는 경로를 설명하지 못하거나, SWO/ITM 출력에만 한정되어 "비동기 로그 출력"이라는 표현이 기술적으로 부정확해진다.
- **권고**: Ch02의 "링 버퍼 기반 비동기 로그 출력" 항목을 Ch05 UART DMA 이후로 이동하거나, Ch02에서는 SWO/ITM 동기 로그까지만 다루고 링 버퍼 + 비동기 출력은 Ch05 또는 Ch12 CLI 챕터에서 통합할 것.

## 🟡 Major 이슈 (비효율적 배치, 중요 내용 누락)

### M-1. Ch06 Complementary PWM & Dead-time — NUCLEO-F411RE 핀 제약 미언급
- **문제**: TIM1의 Complementary 출력(CH1N/CH2N/CH3N)은 NUCLEO-F411RE에서 PA7(TIM1_CH1N), PB0(TIM1_CH2N), PB1(TIM1_CH3N)에 매핑된다. 이 핀들은 Arduino 헤더에서 다른 기능과 충돌할 수 있으며, 스텝 모터(Ch08)·LCD SPI(Ch10) 핀 할당과 사전에 검증되어야 한다.
- **권고**: 목차 또는 챕터 도입부에 전체 핀 할당 테이블(Pinout Table)을 별도 부록이나 Part 0에 명시할 것.

### M-2. Ch07 RTC — LSE 크리스탈 하드웨어 탑재 여부 확인 필요
- **문제**: NUCLEO-F411RE 기본 보드에는 PC14/PC15에 LSE(32.768 kHz) 크리스탈이 **기본 미장착**이다(SB48/SB49 솔더 브리지 설정 필요). 목차에서 "CubeMX 설정 (LSE vs LSI)"를 언급하나, LSE 미장착 시 LSI(~32 kHz, 정밀도 낮음)로 대체해야 한다는 점이 명시되어 있지 않다.
- **영향**: 학습자가 LSE를 설정해도 RTC가 동작하지 않는 하드웨어 트러블이 발생할 수 있다.
- **권고**: Ch07에 NUCLEO-F411RE LSE 장착 여부 확인(UM1724 User Manual 참조) 및 LSI 대체 사용 시 정밀도 편차(±5%) 경고 내용을 포함할 것.

### M-3. Ch08 스텝 모터 — "아날로그 시침 구동"과 28BYJ-48 사양 정합성
- **문제**: ULN2003으로 구동하는 표준 유니폴라 스텝 모터는 통상 28BYJ-48(기어드, 실제 스텝 각 약 0.088°/스텝 after gear)이다. 이를 "아날로그 시침"으로 사용하려면 기어비, 풀 스텝/하프 스텝 시 실제 분해능 계산, 1회전 스텝 수(4096 하프 스텝)에 대한 기술적 근거가 챕터에 포함되어야 한다.
- **영향**: "RTC 시각 → 침 각도 변환 알고리즘"이 기어비 없이는 수식을 제시할 수 없다.
- **권고**: 스텝 모터 사양(28BYJ-48 기어비 64:1, 스트라이드 각 5.625°/스텝, 실제 1회전 = 2048 풀 스텝 또는 4096 하프 스텝)을 Ch08 핵심 주제에 명시할 것.

### M-4. Ch14 소프트웨어 아키텍처 — Part 5 배치가 너무 늦음
- **문제**: FSM, 레이어드 아키텍처, 시퀀스 다이어그램은 Ch14에서 처음 체계적으로 등장하나, Ch08(스텝 모터 FSM), Ch12(CLI 시퀀스 다이어그램), Ch13(레이어 분리) 등에서 이미 동일 개념이 선행 사용된다. "v2.0 전체 리팩토링" 시점이 너무 늦어 앞선 챕터의 코드 품질이 일관되지 않을 위험이 있다.
- **권고**: FSM 기본 개념과 레이어드 아키텍처 개요는 Part 0 또는 Part 1 초반(Ch01~Ch03 사이)에 간략히 소개하고, Ch14에서 심화·통합 리팩토링으로 구성하는 "복선 구조"를 권장한다.

### M-5. 인터럽트 우선순위(NVIC) 및 임계 구역(Critical Section) 챕터 누락
- **문제**: DMA(Ch04~Ch05), EXTI(Ch01), UART IT(Ch03), RTC 알람(Ch07), I2C DMA(Ch11) 등 다수 인터럽트 소스가 혼용되나, NVIC PreemptPriority / SubPriority 설정, HAL_NVIC_SetPriority, __disable_irq() / taskENTER_CRITICAL() 등 인터럽트 충돌 방지 전략을 다루는 챕터가 없다.
- **영향**: 다중 인터럽트 환경에서 우선순위 역전(Priority Inversion), 재진입(Reentrancy) 문제가 발생할 수 있으며 Ch16에서 "재진입 문제"가 한 줄로만 언급된다.
- **권고**: Ch04 DMA 또는 Ch05 UART DMA 챕터에 NVIC 우선순위 설정 절을 추가하거나, Part 1 말미에 "인터럽트 관리 & NVIC" 챕터를 신설할 것.

### M-6. Ch13 PC 대시보드 — 임베디드 강의 범위 이탈 가능성
- **문제**: Python + PyQt5/tkinter + pyserial 기반 PC 애플리케이션 개발은 임베디드 C 실무 강의 범위를 벗어난다. 4시간 중 PC 소프트웨어에 할애하는 시간이 MCU 측 프로토콜 설계보다 많아질 위험이 있다.
- **권고**: MCU 측 직렬 프레임 구조(헤더/페이로드/체크섬) 설계 및 구현에 중점을 두고, PC 대시보드는 제공된 예제 코드 수준으로 한정하거나 선택 실습으로 분리할 것.

## 🟢 Minor 이슈 (보완하면 좋은 것)

### m-1. Ch03 UART — HAL_UART_Receive_IT의 1바이트 수신 한계 미언급
- HAL_UART_Receive_IT는 지정 바이트 수 수신 후 콜백이 호출되는 구조로, 가변 길이 메시지 수신 시 매 바이트 재등록 패턴이 필요하다. 이 패턴(1바이트씩 재등록 후 직접 버퍼 관리)은 실무에서 자주 오해를 유발한다. Ch03 또는 Ch05에서 명시적으로 다룰 것.

### m-2. Ch09 I2C — SHT31 CRC-8 다항식 명시 권장
- SHT31 CRC 검증은 다항식 0x31(x^8 + x^5 + x^4 + 1), 초기값 0xFF를 사용한다. 코드 예제에서 이를 명시하지 않으면 학습자가 임의 CRC 구현 오류를 디버깅하는 데 불필요한 시간을 소모한다.

### m-3. Ch10 ILI9341 — SPI 클럭 속도 상한 명시 권장
- ILI9341의 SPI Write 최대 클럭은 데이터시트 기준 66 MHz이나, NUCLEO-F411RE APB2 클럭(최대 100 MHz)에서 SPI1 프리스케일러 설정 시 실제 안정 동작 속도(일반적으로 42 MHz 이하 권장)를 명시할 것. 너무 빠른 클럭은 ILI9341 초기화 실패로 나타나 디버깅이 어렵다.

### m-4. Ch11 I2C DMA — HAL I2C DMA의 known limitation 언급 권장
- STM32F4 HAL I2C DMA에는 Repeated Start 조건 처리 미지원 이슈가 있다. SHT31처럼 Write 후 Repeated Start로 Read 하는 센서는 DMA 모드에서 별도 구현(직접 레지스터 또는 상태 머신)이 필요하다. 목차 핵심 주제에 이 제약을 명시할 것.

### m-5. Ch15 디버깅 — OpenOCD / ST-LINK GDB 서버 버전 호환성 언급 권장
- STM32CubeIDE 버전에 따라 내장 ST-LINK GDB 서버와 펌웨어 버전 불일치로 연결 실패가 빈번하다. Ch15에 ST-LINK 펌웨어 업그레이드 절차(STM32CubeProgrammer)를 포함할 것.

### m-6. 전체 목차 — 핀 할당 충돌 방지를 위한 마스터 Pinout 부록 없음
- 16개 챕터에 걸쳐 GPIO, UART, SPI, I2C, TIM, RTC, 스텝 모터 핀이 누적 할당된다. 핀 충돌 여부를 학습자가 확인할 수 있는 마스터 Pinout 테이블(부록 또는 Part 0)이 없다.

## ✅ 잘 된 점

1. **누적 성장형 프로젝트 구조가 탁월하다.** v0.1 → v1.3 → v2.0 버전 관리를 목차 수준에서 명시하여 학습자가 각 챕터 완료 후 동작하는 결과물을 가질 수 있다. 임베디드 교육에서 흔히 빠지는 "챕터마다 별개 예제" 함정을 피했다.

2. **DMA 아키텍처(Ch04)와 응용(Ch05, Ch10, Ch11) 분리 전략이 좋다.** 이론 챕터를 먼저 두고 UART·SPI·I2C에 순차 적용하는 구성은 선수 지식 의존 관계를 잘 반영했다.

3. **Ch02 로그 시스템을 초반에 배치한 것은 실무적 판단이다.** SWO/ITM + SEGGER RTT + 로그 레벨 매크로를 Ch02에서 확립하면 이후 모든 챕터의 디버깅 품질이 향상된다. 단, 링 버퍼 비동기 출력 항목은 Critical 이슈 C-3에서 지적한 대로 시점 조정이 필요하다.

4. **Ch08 스텝 모터와 RTC 연동이 프로젝트 목적에 부합한다.** 스마트 시계 프로젝트에서 아날로그 시침을 RTC로 구동하는 구성은 타이머·RTC·모터 제어를 의미 있게 통합한다.

5. **Ch12 CLI와 Ch13 PC 대시보드의 순서가 적절하다.** MCU 측 명령어 파서를 먼저 완성한 후 PC 연동으로 확장하는 흐름이 자연스럽다.

6. **Ch15~Ch16을 Part 6으로 후반에 배치한 것은 타당하다.** 디버깅 심화와 코드 품질 개선은 실제 코드가 충분히 쌓인 후 수행할 때 학습 효과가 크다.

7. **MISRA-C, HAL Mock 단위 테스트, HardFault 스택 덤프 등 실무 수준 주제가 포함되어 있다.** 단순 예제 수준을 넘어 현업 코드 품질 기준을 다루는 점이 중급 대상 강의로서 적합하다.
