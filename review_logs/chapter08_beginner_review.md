## 용어 점검

### 첫 등장 + 한글 설명 ✅ 충분한 용어
- Complementary PWM (상보 출력) — §8.0에서 첫 등장, 괄호 설명 있음
- Dead-time — §8.1에서 "둘 다 꺼지는 안전 구간"으로 명확 설명
- Input Capture (입력 캡처) — §8.0에서 첫 등장
- Output Compare (출력 비교) — §8.4에서 설명
- One-Pulse (원 펄스) — §8.4에서 "트리거 한 번에 펄스 딱 하나" 설명
- TRGO (Trigger Output) — §8.5에서 설명
- FSM (유한 상태 머신) — Ch03 복습 명시
- shoot-through — §8.1에서 "전원에서 GND로 직통" 설명

### 첫 등장 한글 설명 △ 부족한 용어
- **MOE (Main Output Enable)** — 표에서 "안전 잠금"이라고 적혀 있으나, 왜 GP Timer에는 없고 TIM1에만 있는지, 실제로 어떻게 동작하는지(비트를 세트해야 출력이 나옴) 본문 설명 부족. §8.2 실습 팁에서 "Automatic Output State Enable" 언급으로 보완되지만, 처음 만나는 독자는 혼란 가능
- **BDTR (Break and Dead-Time Register)** — 레지스터 이름만 등장, "Break와 Dead-time을 함께 관리하는 레지스터"라는 한 줄 설명 추가 권장
- **DTG (Dead-Time Generator)** — "DTG 비트 필드"로만 등장, "DTG = Dead-Time Generator의 약자, BDTR 레지스터 안의 비트 필드" 보충 권장
- **Break 입력 (BKIN)** — 표에서 "비상 정지 핀"으로 적혀 있으나, 어떤 상황에서 쓰는지 한 문장 보충 권장
- **Slave Mode (종속 모드)** — §8.5 표에서 Trigger/Reset/Gated 모드 설명은 있으나, "Slave Mode"라는 용어 자체의 한글 풀이 누락
- **ITR (Internal Trigger)** — "내부 연결선(ITR)"으로 언급되나, ITR0~ITR3이 구체적으로 어떤 타이머 쌍인지 미제시

### 미설명 용어
- **H-Bridge / Half-Bridge** — "하프 브리지(Half-Bridge) 회로"로 등장하지만, H-Bridge와 Half-Bridge의 차이는 미설명. 중급 개발자가 회로 경험이 없을 수 있으므로 1~2문장 보충 권장
- **FET (트랜지스터)** — "FET(트랜지스터)"로 병기되어 있으나, MOSFET과의 관계 미설명. 단, 이 교재의 범위를 넘으므로 현재 수준 적절

## 코드 이해도

### 가독성: ✅ 양호
- 함수명이 목적 명확: `complementary_pwm_demo()`, `input_capture_start()`, `stopwatch_handle_event()`
- 변수명 C99 스네이크 케이스 준수: `s_capture_1st`, `s_elapsed_ms`, `s_tick_count`
- 한국어 주석 충실, LOG 메시지로 동작 추적 가능

### 주석 충분도: ✅ 양호
- 각 코드 블록에 목적 주석 있음
- FSM switch-case 내 각 전이에 `/* IDLE → RUNNING: 시작 */` 형태 주석

### 30줄 이하 원칙: ✅ 준수
- ch08_01 (CPWM): ~15줄
- ch08_02 (IC): ~30줄 (콜백 포함, 적정)
- ch08_05 (FSM): ~65줄이지만 헤더+구현 합산이므로 개별 함수는 30줄 이하

### ❌ 치명적 버그 발견 (§8.7 main.c 코드, 1046~1052행)

```c
if (duration >= 2000) {
    stopwatch_handle_event(SW_EVT_BTN_LONG);
} else {
    stopwatch_handle_event(SW_EVT_BTN_LONG);  // ← 여기! BTN_SHORT이어야 함
}
```

**두 분기 모두 `SW_EVT_BTN_LONG`을 전달합니다.** `else` 분기는 `SW_EVT_BTN_SHORT`이어야 합니다. 이대로 실행하면 짧게 누르든 길게 누르든 항상 Long 이벤트가 발생하여 스톱워치가 정상 동작하지 않습니다.

**심각도: 높음** — 독자가 코드를 그대로 복사하면 동작하지 않음. 반드시 수정 필요.

### △ 버튼 duration 측정 로직 의문 (§8.7 main.c)

현재 코드에서 `g_btn_press_ms`는 EXTI 콜백(Falling Edge? Rising Edge?)에서 한 번만 기록됩니다. 그런데 `duration = HAL_GetTick() - g_btn_press_ms`는 "버튼을 누른 시점부터 메인 루프에서 처리하는 시점까지의 시간"이지, "버튼을 누르고 있는 시간"이 아닙니다. 실제로 Long Press를 구현하려면 Press/Release 두 이벤트를 캡처하거나, 타이머 기반 디바운싱이 필요합니다. 이 부분은 교육 목적상 단순화된 것으로 이해하지만, 실제 동작과 차이가 있음을 aside로 언급해 주면 좋겠습니다.

### 조건부 승인 — 필수 수정 1건, 권장 수정 4건

### 🔴 필수 수정 (1건)

1. **§8.7 main.c 코드 버그 수정** (1048~1051행)
   - `else` 분기의 `SW_EVT_BTN_LONG` → `SW_EVT_BTN_SHORT`로 수정
   - 이 버그가 수정되지 않으면 독자가 코드를 복사해도 동작하지 않음
   - **심각도: 높음**

### 🟡 권장 수정 (4건)

1. **§8.1 MOE 보충 설명** — "TIM1은 안전 목적으로 MOE(Main Output Enable) 비트가 세트되어야 출력이 활성화됩니다. GP Timer는 이 제한이 없어 PWM_Start()만으로 출력됩니다." 1~2문장 추가

2. **§8.5 ITR 매핑 표 추가** — STM32F411RE에서 어떤 타이머가 어떤 ITR로 연결되는지 간단한 표 1개 (예: ITR0 = TIM1, ITR1 = TIM2 등). 없으면 "내부 연결선"이 추상적으로 남음

3. **§8.6 FSM SVG 점선 전이 명확화** — RUNNING→IDLE(BTN_LONG) 전이가 점선으로 표시되어 "선택적" 느낌. 실선으로 변경하거나 범례에 "점선 = 긴급 리셋 경로" 추가

4. **§8.1 Dead-time 제목과 본문 비유 일관성** — 제목은 "에어록(Airlock)"이지만 본문에서는 시소 비유에 통합되어 에어록 비유가 전개되지 않음. 에어록 비유를 1~2문장으로 전개하거나, 제목을 "안전 구간"으로 변경
