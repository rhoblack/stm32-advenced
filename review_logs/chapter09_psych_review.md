# Ch09 교육심리전문가 의견서

## 1. 감정 곡선 평가

| 구간 | 목표 감정 | 달성 | 평가 |
|------|----------|------|------|
| §9.0 도입 | 호기심 | **O** | "타이머 앱은 있는데 시계 앱이 없는 셈" — 핸드폰 비유가 즉각적 공감 유도. Ch08 성취 확인("여러분이 직접 설계한 첫 FSM") 문장도 있어 자기효능감 보강됨 |

| §9.1 비유 | 안심 | **O** | 전광판 비유 + "디버거에서만 BCD 읽으면 됨" 마무리가 효과적. 비유 한계("메모리에서는 A~F 저장 가능") 명시도 신뢰 구축 |

| §9.4 알람 | 호기심 | **O** | "스마트폰 알람 앱과 정확히 같은 원리" — 생활 경험 연결 우수. 마스크 표가 시각적으로 명확 |

## 2. 학습 불안 5포인트 완화 평가

### 불안 1: BCD 낯선 구조
- **상태**: **완화됨 (O)**
- 전광판 비유 즉시 제공 (§9.1 line 182~187)
- "HAL이 자동 변환하므로 실무에서 직접 BCD 계산할 일은 거의 없습니다" (line 240) — 부담 해제
- 강사 꿀팁: "칠판에서 1~2회만 연습하고 빠르게 넘어가세요" (line 479) — 시간 투자 상한 제시
- BCD→Binary 변환 코드가 3줄로 간결 (line 206~213)

### 불안 2: LSE vs LSI 선택 혼란
- **상태**: **완화됨 (O)**
- "오늘 실습에서는 LSI를 사용합니다" (line 310) — 명확한 선택지 제공
- SB48/SB49 하드웨어 확인 방법 명시 (line 299~307)
- ±5% 오차의 실제 영향 계산: "4시간 동안 약 12분 오차 — 동작 확인에 지장 없습니다" (line 312) — 수치로 안심
- 비교표(line 283~294)가 판단 기준 제공

### 불안 3: Alarm vs Wake-up 차이
- **상태**: **완화됨 (O)**
- "알람 시계 vs 주방 타이머" 비유 (line 623~627) — 1문장 구분
- FAQ에서 "언제 뭘 쓰나요?" 직접 답변 (line 718~723)
- 프로젝트에서 둘 다 사용 예고: "Alarm으로 사용자 알람, Wake-up으로 주기적 센서 체크"
- TIM vs Wake-up 비교표 (line 642~652)로 3자 비교까지 완료

### 불안 4: RTC 콜백 흐름
- **상태**: **완화됨 (O)**
- "Ch08에서 배운 패턴 그대로입니다" (line 584) — 기존 지식 연결 명시
- "EXTI 콜백 → 플래그 → main loop"가 "RTC 콜백 → 플래그 → main loop"로 바뀐 것 (line 585~588)
- **GetTime+GetDate 쌍 호출**: "100% 모든 수강생이 처음에 빠지는 함정" (line 358) + 핵심 정리에서 재강조 (line 1198) + 실습 트러블슈팅 (line 1182) = **3회 이상 강조 달성**

### 불안 5: v0.9 = v0.8 + 추가
- **상태**: **완화됨 (O)**
- "교체가 아닌 병렬 추가" (line 80)
- "v0.8의 모든 것이 그대로 유지됩니다. 교체가 아닌 추가입니다" (line 1031)
- main.c에서 stopwatch_service #include + 초기화 유지 코드 (line 1040, 1061)
- 컴포넌트 목록표에서 기존 컴포넌트와 신규(★) 명확 구분 (line 1101~1115)

**학습 불안 종합**: 5개 핵심 불안 포인트 모두 완화 전략 존재. 특히 "택시"(HAL이 해줌) 전략과 "패턴 반복" 전략이 동시에 작동.

## 6. 최종 권고

### 필수 수정 (0개)
- 없음. 교육심리 관점에서 필수 수정 사항 없음.

### 권장 수정 (2개)

1. **§9.5~9.6 사이 메타인지 박스 추가 고려**: §9.5 Wake-up 이후 §9.6 Alarm FSM 진입 전에 "여기까지 배운 RTC 기능(시간읽기, 알람, Wake-up)을 정리할 수 있는가?" 형태의 점검 박스가 있으면 §9.6 FSM 설계 진입 시 인지 부하가 줄어듦. 다만 현재도 충분히 양호하므로 선택적.

2. **§9.7 "첫 번째 FSM은 어려웠지만, 두 번째는 쉽다" 명시적 확인 문장**: line 1004 근처에서 "Ch08에서 FSM을 처음 구현할 때의 긴장감이 기억나시나요? 지금 이 코드를 보면서 '이건 할 수 있겠다'는 느낌이 든다면, 그것이 패턴 학습의 증거입니다" 같은 감정 확인 문장이 있으면 자기효능감이 더 강화됨.

### 특별 칭찬 사항

1. **"100% 모든 수강생이 처음에 빠지는 함정입니다"** (GetTime+GetDate) — 실패 정상화의 교과서적 사례. 실패 전에 미리 알려주면서도 "일부러 경험해 보세요"로 능동적 학습 유도.

2. **Stopwatch↔Alarm 1:1 대응표** (§9.7) — 패턴 전이(transfer) 학습의 핵심 도구. "이름만 바꾸면 된다"는 것을 표로 증명하여 학습자가 "나도 할 수 있다"를 시각적으로 확인.

3. **"교체가 아닌 추가"** 반복 — 누적 성장 모델에서 가장 큰 불안은 "기존 것이 깨질까?"인데, 이를 3회 이상 반복 강조하여 완전히 해소.
