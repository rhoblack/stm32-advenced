## 비유 검증

### 비유 1: "커피숍 직원" (1절)

**기술적 매핑 검증**:
- "머신이 추출하는 동안 직원이 기다린다" = CPU가 I2C 완료를 폴링으로 대기 ✅
- "완료 신호를 울리면 그때 가져간다" = DMA TC 인터럽트 → 콜백 호출 ✅
- 비유의 한계 명시: "직원이 두 명인 것에 가깝다" ✅

**평가**: 기술적으로 정확하고 이해하기 쉬운 비유. 강의에서 그대로 사용 가능.

### 비유 2: "고속도로 차선" (4절)

**기술적 매핑 검증**:
- "스트림 = 차선, 동시에 차 한 대만 달릴 수 있다" = DMA 스트림은 한 번에 하나의 요청만 처리 ✅
- 비유의 한계 명시: "2차원 구조(스트림+채널)" ✅

**평가**: 스트림 개념을 전달하기에 적절. 강의에서 화이트보드에 차선 그림을 그리면서 설명하면 효과적.

### 비유 3: "택시 환승 vs 새 택시" (3절)

**기술적 매핑 검증**:
- "STOP 없이 방향을 바꾼다" = Repeated Start에서 버스 점유권 유지 ✅
- "단일 마스터에서 차이가 없다" ✅

**평가**: 적절한 비유. 단, "다른 마스터가 끼어들 틈이 없다"는 설명이 교실에서 혼란을 줄 수 있습니다. 수강생이 "그럼 우리 프로젝트에서 다른 마스터가 있나요?"라고 질문할 것입니다. "단일 마스터 환경에서는 실질적 차이 없음"을 더 강조하는 것이 좋겠습니다.

## 수강생 막힘 포인트 현장 검증

### 막힘 1: "콜백이 불리지 않음" (5절 FAQ에서 다룸)
현장에서 가장 자주 발생하는 문제. FAQ에서 3가지 원인을 제시했지만, 실제 현장에서는
**④ i2c DMA 핸들(hdma_i2cx_rx 등)을 I2C 핸들에 연결하지 않은 경우**도 흔합니다.

CubeMX가 MX_I2C1_Init()에서 자동으로 `__HAL_LINKDMA(&hi2c1, hdmarx, hdma_i2c1_rx)`를
생성해야 하는데, DMA Request가 추가되지 않으면 이 코드가 없습니다.
이 원인을 FAQ에 추가하면 더 완전한 체크리스트가 됩니다.

### 막힘 2: "HAL_BUSY 에러" (3절 팁에서 다룸)
현장에서 두 번째로 자주 나오는 문제. 타이머 해법이 제시되어 있습니다. ✅

**강사 추가 팁**: 이 오류를 디버깅할 때 `hi2c1.State`와 `hi2c1.PreviousState` 변수를 Watch Window에서 확인하도록 안내하면 독자가 스스로 진단할 수 있습니다.

### 막힘 3: "TIM6 설정이 왜 PSC/ARR 그 값이냐" (기술 리뷰에서 오류 발견)
현장에서 TIM 계산은 항상 수강생이 직접 계산해보길 원합니다.
교재에 계산 과정을 명시하면 수강생이 "자기 환경(다른 클럭)에서 어떻게 바꿔야 하나"를 스스로 해결할 수 있게 됩니다.

`TIM6 클럭 = 84MHz, 목표 = 15ms 주기`
`PSC=839 → 타이머 클럭 = 84MHz/840 = 100kHz`
`ARR=1499 → 주기 = 1500 × (1/100kHz) = 15ms` ✅

이 계산 과정을 코드 주석 또는 설명에 추가해야 합니다.

**추가 권장**:

"ISR(인터럽트 서비스 루틴)에서 float 연산을 수행해도 되는지, 그 이유를 설명하세요."

## 4시간 강의 흐름 평가

| 시간 | 내용 | 적합성 |
|------|------|--------|
| 0~15분 | Ch11 폴링 코드 복습 + DMA 전환 동기 (1절) | ✅ 적절 |
| 15~60분 | HAL I2C DMA API + CubeMX 설정 (2절) | ✅ 적절 (데모 포함) |
| 60~105분 | Repeated Start 우회 + 콜백 체인 (3절) | ⚠ 45분이 다소 촉박, 60분 권장 |
| 105~150분 | DMA 스트림 충돌 + 우선순위 (4절) | ✅ 적절 |
| 150~210분 | 실습 (Step 1~3) | ✅ 적절 |
| 210~240분 | 5절 연속 루프 + 핵심 정리 + 연습문제 | ✅ 적절 |

**총평**: 4시간(240분) 내에 수행 가능하지만, 3절(Repeated Start + 콜백 체인)이 가장 어려운 구간이므로 이 부분에서 시간 여유를 두는 것이 좋습니다. 필요시 4절 DMA 충돌 분석의 일부를 과제로 돌릴 수 있습니다.
