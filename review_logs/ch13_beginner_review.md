- C언어 숙련, HAL 기본 사용 경험 있음

- GFX 레이어, FSM 테이블 방식은 처음 접함
- ILI9341은 Ch12에서 초기화까지 완료한 상태

## 절별 이해도 평가

### 도입부 (아키텍처 위치, FSM, 시퀀스)
**이해도**: ⭐⭐⭐⭐
- 아키텍처 다이어그램이 Ch13에서 추가되는 부분을 주황 점선으로 강조하여 한눈에 파악 가능 ✅
- 인터페이스 API 목록이 깔끔하게 정리됨 ✅
- FSM 다이어그램에서 BTN_LONG의 "자기 전이" 화살표가 시각적으로 작아서 처음에 놓칠 수 있음 ⚠

### 1절 (GFX 레이어 아키텍처)
**이해도**: ⭐⭐⭐⭐⭐
- 레스토랑 비유가 계층 분리 개념을 직관적으로 설명함 ✅
- Ch12 방식 vs Ch13 방식 비교 SVG가 매우 효과적 ✅
- "LCD 교체 시 gfx_service.c만 수정하면 된다"는 실용적 메시지가 명확 ✅
- FAQ에서 "중복 아닌가요?" 질문이 실제로 많이 나오는 질문을 선점해서 해결함 ✅

### 2절 (폰트 렌더링)
**이해도**: ⭐⭐⭐⭐
- 비트맵 폰트 개념 자체가 중급자에게 새로운 개념이라 처음에는 낯설음
- 비트맵 SVG 다이어그램(ch13_sec02_font_bitmap.svg)이 픽셀 그리드와 바이트 배열을 함께 보여줘서 이해에 큰 도움 ✅
- `0x80 >> col` 비트 연산 설명이 코드 주석으로만 있고 본문에 추가 설명이 없음 ⚠
  → "col=3일 때 0x80>>3 = 0x10 = 0001_0000" 같은 구체적 예시 추가 권장
- 강사 꿀팁(그리드 종이 실습)이 있어 안심됨 ✅

### 3절 (화면 레이아웃)
**이해도**: ⭐⭐⭐⭐⭐
- 레이아웃 SVG(ch13_sec03_layout.svg)가 3종 화면의 좌표를 명확히 보여줌 ✅
- 좌표 상수 전략의 필요성이 "3개월 후 코드" 비유로 잘 설명됨 ✅
- `GFX_FillRect()` 먼저 지우는 패턴 설명이 충분함 ✅
- 신문 편집 비유가 좌표 격자 개념을 쉽게 전달함 ✅

### 4절 (DISPLAY FSM)
**이해도**: ⭐⭐⭐⭐
- Ch08 스톱워치 FSM 복습 언급이 자연스러운 연결고리 역할 ✅
- 테이블 기반 FSM이 코드로 명확히 구현됨 ✅
- 지하철 노선도 비유가 FSM 테이블의 확장성을 잘 설명함 ✅
- "진입 액션 vs 부분 갱신" 분리 설명이 처음에는 헷갈릴 수 있음 ⚠
  → FAQ에서 "방 청소 vs 매일 정리" 비유로 해소됨 ✅
- ISR-safe 패턴의 "왜 ISR에서 직접 호출하면 안 되는가"를 DMA 충돌로 연결하는 설명이 충분함 ✅

### 5절 (DMA 최적화)
**이해도**: ⭐⭐⭐⭐
- "선택 사항" 안내가 명시되어 있어 심리적 부담 감소 ✅
- 숫자(76,800픽셀, 153,600바이트, 63ms)가 구체적으로 제시되어 직관적 이해 가능 ✅
- Dirty Flag 패턴 코드가 명확함 ✅
- `ILI9341_Set_Address_Window` 파라미터 설명에서 "x+w-1" 설명이 충분함 ✅
- Off-by-one 오류를 "흔한 실수"로 정상화한 것이 좋음 ✅

### 6절 (아키텍처 갱신)
**이해도**: ⭐⭐⭐⭐⭐
- 변경 사항 요약 표(테이블)가 한눈에 정리됨 ✅
- 완성 아키텍처 SVG가 전체 그림을 보여줌 ✅

### 실습
**이해도**: ⭐⭐⭐⭐⭐
- 파일 추가 단계(Step 1)가 명확 ✅
- main.c 통합 코드가 완전하여 그대로 따라할 수 있음 ✅
- 로그 출력 예시가 있어 동작 확인 기준이 명확 ✅
- "실습이 안 될 때" 체크리스트가 4가지 케이스를 커버함 ✅

## 발견된 문제점

### [중급 수준 검증]
**Q. Ch12 지식만으로 이 챕터를 이해할 수 있는가?**
- GFX 레이어: ✅ Ch12의 ILI9341 Driver 위에 레이어를 추가하는 개념이므로 연결 자연스러움
- FSM: ✅ Ch08 스톱워치 FSM 복습 언급으로 연결
- 비트맵 폰트: ⚠ 완전히 새로운 개념이지만 SVG와 코드 예시로 충분히 설명됨
- DMA 최적화: ✅ Ch05, Ch12에서 DMA 기초를 배웠으므로 이해 가능

### [설명 없이 등장하는 용어]
- `Bresenham` — 본문에서 "(브레즌험 알고리즘)"으로 한글 설명 병기 ✅
- `Midpoint Circle` — 코드 주석에만 있고 본문 설명 없음 ⚠ (Minor — 도형 함수 설명절에 추가 권장)
- `Glyph` — 코드 주석에 "(문자 모양)"으로 설명됨 ✅
- `Deferred Processing` — 4절 실무 팁에서 설명됨 ✅

## 전체 이해도 평가

**종합 이해도**: ⭐⭐⭐⭐ (4/5)

**강점**:
- 비유와 실세계 예시가 모든 절에 포함되어 있음
- SVG 다이어그램이 개념을 시각적으로 보조함
- "흔한 실수 정상화" 문구가 여러 곳에 삽입되어 심리적 안정감 제공
- FAQ가 실제 수강생 질문을 선점하여 해소

**개선 권장 사항**:
1. 2절에서 `0x80 >> col` 연산 구체적 예시 추가 (col=0,1,3 각각의 결과값)
2. `Midpoint Circle` 알고리즘 이름 옆에 한글 설명 병기
3. FSM 다이어그램의 BTN_LONG 자기 전이 화살표를 좀 더 시각적으로 강조

**승인 기준**: ⭐⭐⭐ 이상 → **조건부 승인** (Minor 개선사항 반영 후 완전 승인)
