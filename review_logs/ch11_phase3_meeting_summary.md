## 🔴 Critical Issues (필수 수정, 2건)

### 1. SHT31 응답 프레임 인덱스 모호성
**출처**: 기술 리뷰어 (기술)
**심각도**: 🔴 Critical
**영향**: CRC 검증 신뢰도, 향후 버그 위험

**현재 상태**:
```c
temp_raw = ((uint16_t)response[0] << 8) | response[1];
humid_raw = ((uint16_t)response[3] << 8) | response[4];
```

**문제**: 인덱스가 매직 넘버 → 유지보수 시 실수 가능
**필수 개선**: 상수 정의 추가
```c
#define SHT31_TEMP_H_IDX    0
#define SHT31_TEMP_L_IDX    1
#define SHT31_TEMP_CRC_IDX  2
#define SHT31_HUM_H_IDX     3
#define SHT31_HUM_L_IDX     4
#define SHT31_HUM_CRC_IDX   5
```

**수정 소요 시간**: 15분
**재검**: 필수 (코드 검토)

---

### 2. HAL I2C 주소 시프트 문서화 부족
**출처**: 기술 리뷰어 (기술)
**심각도**: 🔴 Critical (팀 협업 리스크)
**영향**: 8비트 주소 오버플로우 가능성

**현재 상황**:
```c
i2c_master_transmit(SHT31_ADDR, ...)  // SHT31_ADDR = 0x44 (7비트)
// 내부: (0x44 << 1) = 0x88 (8비트) ✓
```

**문제**: 만약 다른 개발자가 `#define SHT31_ADDR 0x88`(8비트)로 정의하면 `0x88 << 1 = 0x110` 오버플로우
**필수 개선**: 함수 문서화 + 타입 체크
```c
/* i2c_driver.h 상단 */
/**
 * @brief I2C 마스터 송신
 * @param dev_addr I2C 슬레이브 주소 (7비트만! 예: 0x44)
 *        경고: 내부에서 8비트로 변환됨 (dev_addr << 1)
 *        오류: dev_addr=0x88(8비트) → 0x110 오버플로우!
 */
```

**수정 소요 시간**: 10분
**재검**: 필수 (문서화 확인)

## 🟡 Major Issues (강력 권장 수정, 7건)

### [M1] CRC-8 데이터시트 검증 필요
**출처**: 기술 리뷰어
**심각도**: 🟡 Major
**영향**: 센서 데이터 무결성 검증 신뢰도

**권장 수정**: 자체 테스트 코드 추가
```c
// ch11_ex03_crc8.c 하단
void crc8_self_test(void) {
    uint8_t test_data[] = {0xBE, 0x00};
    uint8_t expected = 0xAC;  // 데이터시트 예제
    uint8_t result = crc8_sht31(test_data, 2);

    if (result == expected) {
        LOG_I("CRC8 self-test: PASS");
    } else {
        LOG_E("CRC8 ERROR: expected=0x%02X, got=0x%02X", expected, result);
    }
}
```

**소요 시간**: 20분

---

### [M2] I2C 타임아웃 일관성 (100ms → 1000ms)
**출처**: 기술 리뷼어
**심각도**: 🟡 Major
**영향**: 드문 타임아웃 에러 가능성

**현재**: `sht31_read_measurement()` 타임아웃 = 100ms (다름!)
**권장**: 모두 1000ms로 통일
```c
#define SHT31_I2C_TIMEOUT_MS 1000  // 일관성 확보
```

**소요 시간**: 10분

---

### [M3] 오픈드레인 설명 강화
**출처**: 초급자 독자
**심각도**: 🟡 Major
**영향**: I2C 하드웨어 이해도 부족

**현재 문제**: "LOW를 만들 때만 능동적" → "HIGH는 어떻게?"가 명확하지 않음
**권장 추가**:
```markdown
오픈드레인은 일종의 "협력 시스템"입니다.
마스터와 슬레이브가 동시에 같은 핀을 "잡아당길(pull)" 수 있는데,
아무도 "밀어올리지(push)" 않으므로, 풀업 저항(보통 4.7kΩ)이
자동으로 HIGH로 유지합니다.
```

**소요 시간**: 15분

---

**심각도**: 🟡 Major
**영향**: 불안 해소 부족, 학습 곡선 증가

2. 목적 (데이터 무결성)
3. 흐름도 (다이어그램 또는 텍스트)

5. 테스트 예시

**소요 시간**: 30분
