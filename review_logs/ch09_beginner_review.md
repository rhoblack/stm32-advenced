## 특별히 좋은 점

1. **GetTime+GetDate 쌍 호출 경고**: "100% 모든 수강생이 빠지는 함정" — 강렬한 경고가 기억에 남음. "일부러 경험해 보세요!"까지 추가하여 능동적 학습 유도.
2. **Stopwatch vs Alarm 1:1 대응 표 (§9.7)**: FSM 패턴 동일성을 코드 레벨에서 증명. "패턴이 보이기 시작한다"는 문장이 정확한 타이밍에 등장.
3. **마스크 5단계 표 (§9.4)**: 복잡한 마스크 개념을 "모두 OFF → 모두 ON" 스펙트럼으로 한눈에 정리.
4. **비유 한계 일관 명시**: 모든 비유에 `<small>` 태그로 한계를 밝힘 — 신뢰 형성.
5. **v0.8→v0.9 "교체가 아닌 추가"**: 누적 성장 원칙 준수. 기존 코드 건드리지 않는다는 안심감.

## 개선 제안

### 필수 (0건)
없음.

### 권장 (2건)

1. **§9.5 Wake-up Timer 변수 미선언**: `wakeup_count++` 코드에서 `wakeup_count` 변수 선언부가 없음. 초급자가 "이 변수는 어디서 온 건지?" 혼란 가능. `static volatile uint32_t wakeup_count = 0;` 선언을 추가하거나 주석으로 "전역 변수로 선언됨" 명시 권장.

2. **§9.8 main.c 주석 오류**: L1074에 `/* stopwatch_process();`로 주석 처리되어 있으나, 닫는 `*/`이 같은 줄 뒤쪽 `/* 스톱워치 FSM (병행) */`과 중첩되어 C 주석 문법 오류 가능. 초급자가 그대로 복붙하면 컴파일 에러 발생 우려.
