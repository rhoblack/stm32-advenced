## 이해하기 어려웠던 부분

### 주의 1: 4절 .map 파일 — 첫 등장 시 시각적 가이드 부족
**상황**: .map 파일의 내용이 처음 보는 독자에게 섹션 이름이 낯설 수 있음
**개선 제안**: STM32CubeIDE에서 .map 파일을 어디서 여는지 메뉴 경로를 구체적으로 안내하면 좋겠음
**심각도**: 낮음 — SVG 다이어그램(그림 19-2)이 충분히 보완함

### 주의 2: PRIMASK/BASEPRI — 이 교재에서 처음 등장
**상황**: `__get_PRIMASK()`, `__set_PRIMASK()`가 Ch19에서 처음 등장
**판단**: 7절 면접 질문 Q19에서 BASEPRI가 나오지만 본문에서 PRIMASK만 다뤄 BASEPRI 개념이 면접 답변에만 등장
**결론**: 면접 준비 관점에서 Q19 답변으로 충분히 설명되어 있어 허용 가능

## 이전 챕터 지식만으로 이해 가능한가?

✅ **Yes** — 모든 개념이 Ch01~Ch18에서 다룬 내용 기반
- volatile: Ch05 DMA에서 언급됨
- PRIMASK/Critical Section: Ch05 NVIC 심화에서 소개됨
- .map 파일: Ch03 아키텍처 설계, Ch18 디버깅에서 간접 소개
- HAL Mock: 새로운 개념이나 레이어드 아키텍처(Ch03)를 통해 자연스럽게 이어짐

## 긍정적 평가

1. **비유의 질**: "MISRA-C = 도로교통법", "HAL Mock = 비행 시뮬레이터" — 기술적으로 정확하고 직관적
2. **Before/After 패턴**: 나쁜 코드를 먼저 보여주고 왜 나쁜지 설명 후 좋은 코드 제시 — 학습 순서 최적
3. **면접 Q&A 형식**: 질문 → 답변 → 핵심 키워드 구조가 스스로 공부하기 매우 편함
4. **8절 마무리**: 챕터별 요약 + 성취감 극대화 — 마지막 챕터의 감정적 마무리로 완벽
