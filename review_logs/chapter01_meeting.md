## 1. 리뷰어별 핵심 피드백 요약

### 기술 리뷰어

- **전반 평가**: HAL API 사용 정확, NUCLEO-F411RE 실동작 패턴 부합. 기술 오류는 원고 텍스트와 주석에 집중됨.
- **Critical-01**: `ch01_ex05` 주석 line 85의 `(SysTick = Priority 0, EXTI = Priority 1 이상 권장)` — SysTick HAL 기본값은 Priority **15(최저)**이며 0이 아님. 독자가 NVIC 우선순위 체계를 정반대로 이해하는 치명적 오류.
- **Critical-02**: §1.2 FAQ에서 `GPIO_PIN_13 = 0x2000`의 도출 근거(`2^13 = 8192 = 0x2000`, 13번째 비트) 설명 불완전.
- **Major-01**: ex04 `EXTI15_10_IRQHandler` 주석 코드가 PIN_10~15 전부 나열 — CubeMX 실제 생성 패턴과 불일치, 초보자 오해 위험.
- **Major-02**: ex02 Open-Drain HIGH 시 "미세한 전류" 설명 오류 (PA5에 외부 VCC 풀업 없으므로 완전 High-Z).
- **Major-03**: ex03 무한 대기 루프 타임아웃 처리 강조 부족.
- **Major-04**: §1.5 ISR 황금 법칙 4항 원자성 설명 불완전 — Cortex-M4에서 32비트 이하 aligned 변수 원자성 언급 없음.
- **Minor-03**: ex05 `s_last_press_tick`에 volatile 누락.

### 독자 (중급 임베디드 개발자)

- **전반 평가**: 이해도 ⭐⭐⭐⭐. 비유·코드·FAQ 구조가 탄탄해 처음 읽는 독자도 막히는 구간이 적음.
- **막힌 포인트 1**: §1.1 PMOS/NMOS/VDD 첫 등장 시 괄호 설명 없음.
- **막힌 포인트 2**: §1.4 NVIC 우선순위 그룹의 SubPriority 존재 이유 불명확.
- **막힌 포인트 3**: §lab Step 2에서 `stm32f4xx_it.c` vs `gpio.c` 파일 선택 혼란 — `HAL_GPIO_EXTI_Callback`을 어느 파일에 쓰는지 불명확.
- **개선 제안**: VDD/PMOS/NMOS 용어 첫 등장 시 한 줄 설명, §lab 파일 배치 명확화, 원자성 예시 보강.

### 교육심리전문가

- **전반 평가**: 심리적 안전감 ⭐⭐⭐⭐. 비유-코드-FAQ-메타인지 점검의 4중 구조가 각 절마다 반복되어 학습 리듬이 예측 가능.
- **[필수] §1.4 진입 전 심리적 완충 장치 부재**: EXTI+NVIC 개념 밀도가 다른 절의 2배 이상인데 심리적 완충 문장 없음. 자기효능감 위기 포인트 1위.
- **[필수] `__weak` 설명 후 안심 문구 부재**: 링커 동작 설명이 "링커를 알아야 하나?"는 새 불안 유발 가능성.
- **조건부 승인**: 위 2건 반영 후 최종 승인 권고.
- **[권장]** 실습 Step 2 파일 구조 오리엔테이션, §1.1 MOSFET 진입 시 선언적 요약 문장.

### 교육전문강사

- **전반 평가**: 강의 적합도 ⭐⭐⭐⭐⭐. 현장 즉시 사용 가능 수준.
- **[높음] `HAL_GetTick()` 오버플로우 FAQ 추가 필요**: §1.3 스스로 점검에 질문만 있고 답변 없음. uint32_t 뺄셈의 모듈러 산술 안전성 설명이 FAQ로 필요. 현장에서 반드시 질문이 나오는 항목.
- **[높음] volatile 필요성을 면접 포인트 박스로 격상**: "ISR과 메인 루프 사이 공유 변수에 volatile을 쓰는 이유"는 중견 기업 STM32 포지션 단골 면접 질문.
- **[보통]** CubeMX Regenerate 코드 유실 경고 FAQ 격상, 실습 전 완성 결과물 시연 강사 팁 추가.
- **[낮음]** 연습문제 4-3 RC 필터 회로 스케치 추가.

## 3. Action Items (수정 목록 — 실행 순서)

### [A1] Critical-01: SysTick 우선순위 수치 수정

- **파일**: `ch01_ex05_button_exti_debounce.c` line 85
- **수정**: `(SysTick = Priority 0, EXTI = Priority 1 이상 권장)` → `(SysTick = Priority 15(최저, HAL 기본값), EXTI = Priority 5~10 범위 권장)`
- **추가**: line 82~85 주석 전체를 정확한 우선순위 관계로 재서술. HAL_GetTick() 1ms 오차 가능성 언급.
- **HTML 연계**: §1.5 FAQ의 "우선순위가 15 이하(숫자 ≥ 15)라면 SysTick이 선점하지 못해" 문장 보완.
- ☐ 완료

### [A2] Critical-02: GPIO_PIN_13 비트마스크 도출 논리 추가

- **파일**: `chapter01.html` §1.2 FAQ 박스
- **수정**: 기존 설명에 `0x2000 = 2^13 = 8192, 즉 13번째 비트(0부터 시작)가 1` 도출 과정 추가.
- ☐ 완료

### [A3] Major-01: EXTI15_10_IRQHandler 코드 패턴 개선

- **파일**: `ch01_ex04_button_exti.c` (주석 내 예시 코드)
- **수정**: PIN_10~15 전부 나열 패턴 제거. `HAL_GPIO_EXTI_IRQHandler(GPIO_PIN_13)` 단일 호출로 교체.
- ☐ 완료

### [A4] Major-04: ISR 황금 법칙 원자성 설명 보완

- **파일**: `chapter01.html` §1.5 ISR 황금 법칙 4항
- **수정**: "Cortex-M4에서 32비트 이하 aligned 변수 접근은 원자적" 추가. uint64_t 및 RMW 연산 위험성 병기.
- ☐ 완료

### [A5] 심리-필수-1: §1.4 EXTI+NVIC 진입 전 완충 문장

- **파일**: `chapter01.html` §1.4 `<h2>` 직후
- **추가**: "이 절에서 다루는 EXTI와 NVIC는 처음 접하면 낯설게 느껴지는 것이 정상입니다. 비유와 그림을 통해 차근차근 따라오시면 반드시 이해할 수 있습니다." 완충 문장.
- ☐ 완료

### [A6] 심리-필수-2: `__weak` 설명 후 안심 문구

- **파일**: `chapter01.html` §1.4 `__weak` FAQ 답변 끝
- **추가**: "지금은 '같은 이름의 함수를 만들면 자동으로 내 코드가 호출된다'는 것만 기억하면 충분합니다."
- ☐ 완료

### [A7] 독자-1: VDD/PMOS/NMOS 첫 등장 시 한국어 설명

- **파일**: `chapter01.html` §1.1
- **수정**:
  - `VDD` → `VDD(전원 공급 전압, 3.3V)`
  - `PMOS` → `PMOS(P채널 금속산화물 반도체 트랜지스터, 전원 쪽 스위치)`
  - `NMOS` → `NMOS(N채널 금속산화물 반도체 트랜지스터, GND 쪽 스위치)`
- ☐ 완료

### [A8] 독자-2: §lab Step 2 파일 배치 명확화

- **파일**: `chapter01.html` §lab `<h4>stm32f4xx_it.c 또는 gpio.c — EXTI 콜백 구현</h4>` 아래
- **추가**: "HAL_GPIO_EXTI_Callback 함수는 main.c 파일 하단 또는 별도 gpio_handler.c 파일에 작성합니다. stm32f4xx_it.c에는 작성하지 마세요."
- ☐ 완료

### [A9] 강사-1: HAL_GetTick() 오버플로우 FAQ 추가

- **파일**: `chapter01.html` §1.5 스스로 점검 또는 FAQ

- ☐ 완료

- **파일**: `chapter01.html` §1.5 volatile 비유 또는 ISR 황금 법칙 근처
