## 2. 비유 검증 (메트로놈 / 기어 / 수도꼭지)

### 비유 A: 메트로놈 (§7.1) — ★★★★★ 정확도 매우 높음
- **매핑**: PSC = 속도 조절, ARR = 한 주기 끝, UEV = "딱" 소리 → 정확
- **한계점 명시**: "실제 메트로놈은 소리만 내지만, STM32 타이머는 PWM 출력, 캡처, 비교 등 다양한 기능" → 명시됨
- **강사 판정**: PASS

### 비유 B: 기어 (§7.2) — ★★★★☆ 기어로 변경됨, 기술적으로 적합
- Phase 1에서 제안한 "물시계"가 "자전거 기어"로 변경됨. PSC = 큰 기어, ARR = 바퀴 둘레, CCR = 바퀴의 표시
- **매핑**: 기어비를 바꾸면 바퀴 속도가 바뀐다 → 분주 개념과 잘 맞음
- **한계점 명시**: "자전거 기어는 연속적이지만 PSC/ARR은 정수 값만 가능" → 명시됨
- **강사 의견**: 물시계보다 기어가 더 직관적. CCR을 "바퀴의 표시"로 설명한 것은 약간 어색하지만 수용 가능
- **강사 판정**: PASS

### 비유 C: 수도꼭지 (§7.4) — ★★★★☆ 신호등에서 수도꼭지로 변경
- Phase 1에서 제안한 "신호등"이 "수도꼭지"로 변경됨. 수도꼭지 열림 비율 = Duty Cycle
- **매핑**: 50% 열면 절반 유량 → 50% PWM이면 절반 밝기 → 직관적
- **한계점 명시**: "수도꼭지는 연속적 유량 조절이지만, PWM은 켜짐/꺼짐 반복" → 명시됨
- **강사 의견**: 한계점을 정확히 짚어줌. PWM의 본질(디지털 on/off 반복)과 비유의 차이를 수강생이 인지할 수 있음
- **강사 판정**: PASS

### 비유 D: 컨베이어 (입력 캡처) — 미사용
- Phase 1에서 맛보기로 제안했으나, Ch07 범위에서 입력 캡처를 다루지 않으므로 적절히 생략됨
- Ch08에서 활용 가능

**비유 종합 판정**: 3/3 PASS. 모든 비유에 한계점이 `<small>` 태그로 명시되어 있어 오해 방지 충분

## 4. 시연 가능성 평가

### 실습 1: TIM2 인터럽트 LED 토글 (§7.3)
- **시연 가능**: PA5(NUCLEO LD2)로 500ms 점멸 → 눈으로 확인 가능
- **주기 변경 데모**: 100ms → 500ms → 1s 변경은 육안으로 차이가 명확
- **판정**: PASS

### 실습 2: PWM LED 밝기 4단계 (§7.4)
- **시연 가능**: PA0에 외부 LED 연결 → 25/50/75/100% 밝기 차이 확인
- **주의**: NUCLEO 내장 LED(PA5)와 PWM(PA0) 핀이 다름 → 외부 LED 필요
- PA5에 TIM2_CH1 매핑 가능하다는 실무 팁이 있으나, 충돌 주의사항도 명시됨
- **판정**: PASS (외부 LED 준비 안내 충분)

### 실습 3: 누적 통합 (§7.6)
- **시연 가능**: 버튼(PC13) → 모드 전환 → LED 속도+밝기 동시 변경
- **강의 임팩트**: "while(1)에 HAL_Delay가 하나도 없다!" → 강력한 시각적 데모
- **판정**: PASS

### 오실로스코프/로직 애널라이저 시연
- **언급 없음**: Phase 1에서 "라이브 데모 필수"로 권고했으나, 원고에 오실로스코프 시연 안내가 없음
- **권장**: §7.4 PWM 절에 "오실로스코프로 1kHz PWM 파형을 확인하면 CCR/ARR 관계가 즉시 이해됨" 강사팁 추가
- **판정**: 조건부 PASS (권장 수정)

## 7. 강점 + 개선점

### 강점 (5개)
1. **"HAL_Delay는 감옥" 서사**: 도입부터 마지막 통합 데모까지 일관된 메시지 — 수강생이 타이머를 배우는 "이유"를 체감
2. **PSC/ARR 계산 표**: 5개 조합의 역산 예제가 공식을 체화시킴. `ch07_01_tim_period_calc.c` 검증 코드까지 제공
3. **uart_driver ↔ tim_driver 패턴 비교표**: 아키텍처 일관성을 시각적으로 증명. "새 드라이버는 같은 패턴"이라는 자신감 부여
4. **§7.6 통합 실습**: 버튼 + TIM + UART + 모드 테이블 — 실무에 가까운 완성도. 모드 전환 패턴은 Ch08 FSM의 자연스러운 예습
5. **연습문제 6번**: 시나리오 기반 타이머 선택 — 블룸 "평가" 수준까지 도달. 면접 대비에도 직결

### 개선점 (3개)

#### [권장-1] §7.4에 오실로스코프 시연 강사팁 추가
- **현상**: PWM 파형을 눈으로 확인하는 방법 안내가 없음
- **제안**: §7.4 PWM 절에 `<aside class="instructor-tip">` 추가
  - "오실로스코프 또는 로직 애널라이저로 PA0의 PWM 파형을 측정하면, CCR을 바꿀 때 High 구간이 실시간으로 변하는 것을 볼 수 있습니다. 장비가 없다면 STM32CubeIDE의 Live Expression으로 TIM2->CCR1 값을 모니터링하세요."
- **심각도**: 권장 (강의 임팩트 향상, 필수는 아님)

#### [권장-2] APB1 타이머 클럭 x2 규칙 명시 강화
- **현상**: §7.1에서 "APB1 타이머 클럭은 84MHz"라고만 언급. "APB1 버스 클럭은 42MHz인데 왜 타이머는 84MHz인가?"에 대한 설명이 부족
- **제안**: §7.1 "타이머 클럭" 소절에 다음 추가:
  - "APB1 버스 클럭은 42MHz이지만, APB 프리스케일러가 1이 아닐 때(현재 ÷2) STM32는 타이머에 자동으로 ×2를 적용합니다. 따라서 타이머 클럭 = 42MHz × 2 = 84MHz입니다. 이 규칙은 STM32 전 시리즈에 공통입니다."
- **심각도**: 권장 (면접 포인트이기도 하므로 명확히 설명하면 좋음)

#### [권장-3] TIM1 클럭 수정 필요
- **현상**: §7.1 타이머 계층 표에서 TIM1(Advanced) 클럭이 "APB2 (100MHz)"로 표기
- **문제**: STM32F411RE의 APB2 최대 클럭은 **100MHz가 아닌 84MHz** (HCLK 84MHz, APB2 프리스케일러 ÷1 = 84MHz). 100MHz는 F411의 최대 시스템 클럭이며 APB2 타이머 클럭과 다름
- **제안**: "APB2 (100MHz)" → "APB2 (84MHz)"로 수정
- **심각도**: **필수** — 기술적 오류. 수강생이 TIM1 PSC/ARR 계산 시 잘못된 값을 사용하게 됨
