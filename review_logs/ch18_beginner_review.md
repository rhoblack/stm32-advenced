# Ch18 독자 리뷰 — beginner_reader.md 기준

## 리뷰어: 초보자 독자 (중급 임베디드 개발자 관점)
## 프로필: C언어 숙련, HAL 기본 사용 경험 있음. DMA/HardFault 처음 접함.
## 날짜: 2026-03-17

---

## 전체 이해도 평가

**⭐⭐⭐⭐ (4/5)**

전반적으로 이해하기 쉽게 구성되어 있습니다. 비유가 각 절마다 제공되어 낯선 개념(HardFault, DMA 에러)에 접근하는 데 도움이 됩니다. 4점에서 5점으로 올라가지 못한 이유는 일부 전제 지식(레지스터 직독, CFSR 비트 필드)이 처음 접하는 독자에게는 여전히 버겁게 느껴질 수 있기 때문입니다.

---

## 절별 이해도 평가

### 1절: STM32CubeIDE 디버거 심화 — ⭐⭐⭐⭐⭐

잘 이해됩니다. Watch, Live Expression, Memory View를 실제 사용 방법과 함께 설명하여 바로 따라 할 수 있습니다. "계기판 비유"가 적절합니다. ST-LINK 펌웨어 업그레이드 절차도 단계별로 명확합니다.

### 2절: HardFault 핸들러 — ⭐⭐⭐⭐

**이해된 부분**: PC 값이 왜 중요한지, Disassembly에서 찾는 방법이 명확합니다. "에어백 비유"가 HardFault의 성격을 잘 설명합니다.

**어려운 부분**:
- ASM 스텁 코드(`TST LR, #4`, `ITE EQ`, `MRSEQ` 등)는 어셈블리 미경험자에게 블랙박스처럼 느껴집니다. "왜 이렇게 해야 하는가"에 대한 1~2문장 설명이 있으면 좋겠습니다. (예: "CPU가 어떤 스택을 사용했는지 LR 비트4로 판별합니다")
- CFSR 비트 의미가 많아 처음에 압도됩니다. 본문에서 "가장 흔한 2개(PRECISERR, DIVBYZERO)만 집중하면 됩니다"라고 안내는 하지만, aside 박스로 더 강조하면 좋겠습니다.

**요청**: ASM 스텁 코드 위에 "이 코드를 이해하지 못해도 됩니다 — 복사해서 쓰면 됩니다. 중요한 것은 C 함수에서 PC 값을 읽는 것입니다" 형태의 안심 문구 추가.

### 3절: DMA 디버깅 패턴 — ⭐⭐⭐⭐

**이해된 부분**: 3종 시나리오(HardFault/DMA 재진입/링 버퍼 오버플로우) 구조가 명확합니다. "물류 트럭 비유"가 DMA 비동기성을 잘 설명합니다.

**어려운 부분**:
- LISR 레지스터의 비트 오프셋 계산(스트림별 6비트씩)이 처음 접하는 독자에게 헷갈립니다. Stream0은 비트0~5, Stream1은 비트6~11 식의 테이블이 있으면 좋겠습니다.
- "직접 모드 에러(DMEIF)"가 무엇인지 설명이 없습니다.

**요청**: LISR 비트 오프셋 간단한 테이블 또는 "Stream0 기준이며, 다른 스트림은 RM0383 Table 43 참조" 각주 추가.

### 4절: 로직 애널라이저 — ⭐⭐⭐⭐⭐

장비 없는 독자를 위한 대안(저가 Saleae 호환, $5~10)과 소프트웨어 대안(ST-LINK SWD 타임라인)이 모두 언급되어 좋습니다. SPI/I2C 핀 연결 표가 명확합니다. GND 연결 중요성 강조도 실용적입니다.

### 5절: ITM/RTT 고급 — ⭐⭐⭐⭐

**이해된 부분**: ITM vs RTT 비교 표가 명확합니다. `.noinit` 섹션 설명은 약간 생소하지만 코드 주석으로 보완되어 있습니다.

**어려운 부분**:
- `SEGGER_RTT_MODE_NO_BLOCK_SKIP` vs `SEGGER_RTT_MODE_BLOCK_IF_FIFO_FULL`의 차이를 모르는 독자를 위해 한 줄 설명 추가 필요.
- DWT 사이클 카운터 초기화에서 `0xC5ACCE55` "마법 숫자"에 대한 설명 없음.

**요청**: RTT 채널 모드 2가지 차이를 "SKIP=빠르지만 손실 가능 / BLOCK=느리지만 손실 없음" 형태로 한 줄 요약 추가.

### 6절: 아키텍처 관점 디버깅 — ⭐⭐⭐⭐

"소방서 비유"가 레이어 격리 전략을 직관적으로 이해하게 해줍니다. `LAYER_ASSERT` 매크로가 구체적이어서 바로 적용 가능합니다.
