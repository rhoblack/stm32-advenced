## 🔑 3개 핵심 개념

1. **링 버퍼 (Ring Buffer)**
   - 고정 크기 배열을 원형으로 사용하는 FIFO 자료구조
   - head/tail 모듈로 연산: `(idx + 1) % SIZE`
   - 빈 vs 가득 판별: tail == head (비어있음) vs (head+1)%SIZE == tail (가득참)
   - **CH04 연계**: 간이 링 버퍼(128B) → 정식 모듈(256/512B)

2. **순환 DMA (Circular DMA)**
   - Ch05 Normal Mode 확장: 전송 완료 후 **자동 재시작**
   - UART RX: DMA가 하드웨어 버퍼를 순환 기록
   - IDLE Line 감지: 가변 길이 수신 완료 판단 (HAL_UARTEx_ReceiveToIdle_DMA)
   - **CH05 연계**: Mem-to-Mem → Peripheral DMA

3. **양방향 DMA 통신**
   - **RX**: 순환 DMA + IDLE 감지 + 링 버퍼 (백그라운드)
   - **TX**: 링 버퍼 큐 + DMA 전송 + 완료 콜백 (논블로킹)
   - **FSM**: TX_IDLE ↔ TX_BUSY (RX는 항시 활성)
   - **인터페이스 불변**: uart_driver.h 시그니처 변경 없음 ← **Ch03 아키텍처의 최강 실증**
